## 二、发现的潜在问题

### 🔴 高优先级

#### 问题1：五行数据结构不一致

**严重程度：** 高
**位置：** `result.js` 第 44 行 & `fortune_engine.js` `calculateFiveElements` 函数

**问题描述：**
两个 `calculateFiveElements` 函数返回的数据结构完全不同：

```javascript
// fortune_engine.js 返回格式
{ metal: 0.25, wood: 0.20, water: 0.15, fire: 0.20, earth: 0.20 }

// result.js calculateFiveElements 返回格式
{ yearElement, monthElement, dayElement, hourElement, preferredElement, elementCount }
```

**影响：**
- `result.js` 第 44 行获取 `const fiveElements = result.fiveElements || this.calculateFiveElements(result)`
- 如果使用引擎返回的数据，访问 `fiveElements.preferredElement` 会返回 `undefined`
- 导致 `calculateLuckyDirection` 和 `calculateFoodRecommendation` 失效

**修复方案：**

在 `result.js` 的 `onLoad` 中，转换引擎返回的五行数据：

```javascript
// 获取五行信息并转换格式
let fiveElements = result.fiveElements;
if (fiveElements && typeof fiveElements.metal === 'number') {
  // 从引擎格式转换为 UI 格式
  const elements = {
    '金': fiveElements.metal,
    '木': fiveElements.wood,
    '水': fiveElements.water,
    '火': fiveElements.fire,
    '土': fiveElements.earth
  };
  const minElement = Object.keys(elements).reduce((a, b) => 
    elements[a] < elements[b] ? a : b
  );
  fiveElements = {
    preferredElement: minElement,
    elementCount: elements,
    // 保留原始数据
    raw: result.fiveElements
  };
} else if (!fiveElements) {
  fiveElements = this.calculateFiveElements(result);
}
```

#### 问题2：Canvas 组件缺失导致保存图片失败

**严重程度：** 高
**位置：** `result.js` `saveResult` 方法

**问题描述：**
`saveResult` 方法中调用了 `wx.createCanvasContext('shareCanvas', this)`，但 `result.wxml` 中没有定义对应的 canvas 组件。

**影响：**
- 用户点击"保存结果"按钮时，会触发 `canvasToTempFilePath` 失败
- 控制台报错：`canvasId 'shareCanvas' not found`

**修复方案：**

在 `result.wxml` 中添加 canvas 组件：

```xml
<!-- 在 result.wxml 底部添加 -->
<canvas canvas-id="shareCanvas" style="position: fixed; left: -9999px; width: 300px; height: 400px;"></canvas>
```

#### 问题4：备选牌型分数逻辑问题

**严重程度：** 中
**位置：** `input.js` 第 190-206 行

**问题描述：**
当前逻辑是：如果备选牌型分数 >= 主牌型分数，则调整分数。但这个逻辑在 `onSubmit` 中执行，意味着每次提交都会修改，但数据已经生成完成。

**潜在问题：**
- `alternatives` 数组在引擎中已经排序，第一项即为最高分
- 如果主牌型分数意外低于备选，说明算法本身有问题

**修复建议：**
算法层面应确保主推荐牌型分数最高，而非在调用后手动修正。建议检查 `fortune_engine.js` 中 `recommendPattern` 的排序逻辑。
