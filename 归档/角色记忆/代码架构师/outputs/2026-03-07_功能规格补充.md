## 1. 合盘分析模块

### 1.1 功能概述

提供八字合盘（日主合、日合、月合、年合）和星座合盘的兼容性分析服务，输出量化评分、吉凶判断、改进建议及详细分析报告。

### 1.2 数据模型

#### 1.2.1 出生信息模型（BirthInfo）

```typescript
interface BirthInfo {
  userId: string;           // 用户ID（必填）
  lunarDate?: {            // 农历（可选，自动转换）
    year: number;           // 农历年
    month: number;          // 农历月（1-12）
    day: number;            // 农历日（1-30）
    hour: number;           // 时辰（0-11）
    isLeapMonth: boolean;   // 是否闰月
  };
  solarDate: {              // 公历（必填）
    year: number;           // 公历年（1900-2100）
    month: number;          // 公历月（1-12）
    day: number;            // 公历日（1-31）
    hour: number;           // 小时（0-23）
    minute: number;         // 分钟（0-59）
  };
  gender: 'male' | 'female'; // 性别（必填）
  timezone?: string;         // 时区（默认：Asia/Shanghai）
}
```

#### 1.2.2 合盘类型枚举（SynastryType）

```typescript
enum SynastryType {
  DAY_MASTER = 'day_master',     // 日主合（日干五行对比）
  DAY_PILLAR = 'day_pillar',     // 日合（日柱整体对比）
  MONTH_PILLAR = 'month_pillar', // 月合（月柱整体对比）
  YEAR_PILLAR = 'year_pillar',   // 年合（年柱整体对比）
  ZODIAC = 'zodiac'              // 星座合盘（太阳星座对比）
}
```

#### 1.2.3 合盘请求模型（SynastryRequest）

```typescript
interface SynastryRequest {
  user1: BirthInfo;              // 用户1出生信息（必填）
  user2: BirthInfo;              // 用户2出生信息（必填）
  synastryType: SynastryType;    // 合盘类型（必填）
  analysisDepth?: 'basic' | 'detailed' | 'comprehensive'; // 分析深度（默认：basic）
  includeRecommendations?: boolean; // 是否包含建议（默认：true）
}
```

#### 1.2.4 兼容性评分模型（CompatibilityScore）

```typescript
interface CompatibilityScore {
  overall: number;               // 总体评分（0-100）
  dimensions: {
    personality: number;         // 性格契合度（0-100）

    emotion: number;             // 情感契合度（0-100）
    finance: number;             // 财运契合度（0-100）
  };
  breakdown: {
    favorable: number;           // 吉祥因素数量
    neutral: number;             // 中性因素数量
    unfavorable: number;         // 不吉因素数量
  };

```

#### 1.2.5 合盘分析结果模型（SynastryResult）

```typescript
interface SynastryResult {
```

### 1.4 异常处理

| 错误代码 | 错误类型 | HTTP状态码 | 描述 | 处理建议 |
|---------|---------|-----------|------|---------|
| SYN_001 | INVALID_BIRTH_INFO | 400 | 出生信息无效（日期范围错误、性别缺失等） | 返回具体错误字段，提示正确格式 |
| SYN_002 | INVALID_SYNASTRY_TYPE | 400 | 不支持的合盘类型 | 返回支持的合盘类型列表 |
| SYN_003 | SAME_USER | 400 | 用户1和用户2相同 | 提示需要不同的用户 |
| SYN_004 | DATE_CONVERSION_FAILED | 500 | 日期转换失败（农历/公历互转） | 使用默认日期或提示用户手动输入 |
| SYN_005 | CALCULATION_ERROR | 500 | 合盘计算过程中出现错误 | 记录错误日志，返回通用错误消息 |
| SYN_006 | RATE_LIMIT_EXCEEDED | 429 | 请求频率超限 | 返回重试时间（Retry-After头） |
| SYN_007 | INVALID_USER_ID | 401 | 用户ID无效或不存在 | 提示用户重新登录 |
| SYN_008 | BATCH_SIZE_EXCEEDED | 400 | 批量请求超过限制 | 提示最大批量数量 |

### 1.5 安全校验

#### 1.5.1 输入验证

- **出生信息验证：**
  - 年份范围：1900-2100
  - 月份范围：1-12
  - 日期范围：根据月份和年份验证（考虑闰年）
  - 小时范围：0-23
  - 分钟范围：0-59
  - 性别必须为 'male' 或 'female'

- **合盘类型验证：**
  - 必须为 SynastryType 枚举值之一

- **用户ID验证：**
  - 必须符合 UUID v4 格式或系统定义的ID格式
  - 长度限制：1-64字符

#### 1.5.2 频率限制

- 同一用户每分钟最多调用 10 次
- 同一用户每小时最多调用 100 次
- 批量请求每分钟最多调用 5 次
- IP级别限制：同一IP每分钟最多调用 50 次

#### 1.5.3 权限校验

- 用户必须已登录（验证JWT token）
- 免费用户每日最多调用 5 次详细分析
- VIP用户无调用次数限制

#### 1.5.4 数据隐私

- 合盘历史记录仅用户本人可查询
- 日志中记录请求ID，不记录具体出生信息
- 敏感数据（出生信息）在日志中脱敏处理

#### 1.5.5 防护机制

- 防止 SQL 注入：使用参数化查询
- 防止 XSS：输入输出均进行转义
- 防止 CSRF：使用 CSRF Token
- 请求签名验证（可选）：对敏感操作进行签名验证

## 2. 积分/打卡体系

### 2.1 功能概述

设计完整的用户积分获取、消耗及防作弊机制，支持每日签到、完成测算、分享结果等积分获取方式，以及解锁详细报告、兑换高级服务、会员等级等积分用途。

### 2.7 安全校验

#### 2.7.1 输入验证

- **eventType 验证：** 必须为 PointsEventType 或 PointsConsumeType 枚举值
- **points 验证：** 必须为整数且大于0
- **日期格式验证：** YYYY-MM-DD 格式

#### 2.7.2 防作弊机制

##### 2.7.2.1 签到防作弊

- **IP限制：** 同一IP今日签到超过5次需验证码
- **设备限制：** 同一设备ID今日签到超过3次需验证码
- **时间窗口限制：** 两次签到间隔必须大于1小时
- **User-Agent检测：** 异常User-Agent模式触发风控
- **行为分析：** 签到时间过于规律（如每日固定秒数）标记为可疑
- **地理位置检测：** 跨国/跨大区域短时间内多次签到触发风控

##### 2.7.2.2 积分获取防作弊

- **频率限制：** 同一事件类型每分钟最多调用10次
- **每日上限：** 严格执行各事件的每日获取上限
- **验证码：** 高风险操作（如邀请、观看广告）需人机验证
- **Token验证：** 使用不可预测的请求token，防止重放攻击
- **签名验证：** 客户端对关键参数进行签名，服务端验证

### 3.5 抽取算法

#### 3.5.1 随机抽取算法

```typescript
function drawFortuneRandom(type: FortuneType, seed?: string): Fortune {
  // 1. 获取该类型所有启用的签文
  const fortunes = await getFortunesByType(type, true);
  
  // 2. 使用带种子的随机数生成器
  const rng = seed ? seededRandom(seed) : cryptoRandom();
  
  // 3. 随机选择一个签文
  const index = Math.floor(rng() * fortunes.length);
  
  return fortunes[index];
}
```

#### 3.5.2 日期索引算法

```typescript
function drawFortuneByDateIndex(date: string, type: FortuneType): Fortune {
  // 1. 获取该类型所有启用的签文
  const fortunes = await getFortunesByType(type, true);
  
  // 2. 将日期转换为可预测的序号
  const dateObj = new Date(date);
  const year = dateObj.getFullYear();
  const month = dateObj.getMonth() + 1;
  const day = dateObj.getDate();
  
  // 3. 基于日期生成序号（考虑闰年）
  const baseIndex = (year * 1000) + (month * 100) + day;
  const isLeapYear = (year % 4 === 0 && year % 100 !== 0) || year % 400 === 0;
  const leapOffset = isLeapYear ? 1 : 0;
  
  // 4. 计算最终索引（使用质数模运算保证均匀分布）
  const index = (baseIndex + leapOffset) % fortunes.length;
  
  return fortunes[index];
}
```

#### 3.5.3 频率优化算法

为避免某些签文被过度抽取，实现带权重的随机选择：

```typescript
function drawFortuneWithWeighting(type: FortuneType, userId?: string): Fortune {
  const fortunes = await getFortunesByType(type, true);
  
  // 获取近期抽取历史
  const recentHistory = userId 
    ? await getRecentFortuneHistory(userId, 7) // 最近7天
    : [];
  
  // 计算每个签文的权重（近期抽取过的权重降低）
  const weightedFortunes = fortunes.map(fortune => {
    const drawCount = recentHistory.filter(h => h.fortuneId === fortune.id).length;
    const weight = Math.pow(0.5, drawCount); // 每抽取一次，权重减半
    return { fortune, weight };
  });
  
  // 按权重随机选择
  const totalWeight = weightedFortunes.reduce((sum, item) => sum + item.weight, 0);
  let random = Math.random() * totalWeight;
  
  for (const item of weightedFortunes) {
    random -= item.weight;
    if (random <= 0) {
      return item.fortune;
    }
  }
  
  return weightedFortunes[weightedFortunes.length - 1].fortune;
}
```
