## 一、用户痛点深度剖析

### 1.1 表面痛点
> "转行的人对目标行业一无所知，信息获取成本太高"

### 1.2 真实痛点
1. **信息碎片化**
   - 网上信息散落各处（知乎、公众号、招聘网站）
   - 需要大量时间搜集、整理、筛选

2. **缺乏系统性**
   - 不知道该学什么（技能树不清晰）
   - 不知道学到什么程度（标准不明确）
   - 不知道怎么学（路径不清晰）

3. **没有个性化**
   - 通用建议不适合个人情况
   - 每次咨询都要重新描述背景

4. **没有持续跟踪**
   - 问完就忘
   - 没有阶段性反馈
   - 容易半途而废

## 五、技术架构

### 5.1 整体架构
```
用户 → 前端（网页/小程序）
     ↓
   API 网关
     ↓
  OpenClaw（智能体调度）
     ├─ 转行咨询师（用户画像采集）
     ├─ 行业研究员（行业调研）
     ├─ 能力评估师（能力评估）
     └─ 规划师（转行计划）
     ↓
  飞书多维表格（数据存储）
     ↓
  飞书文档（知识库）
```

### 5.2 智能体调用流程
```
1. 用户开始咨询
   → 转行咨询师 spawn
   → 对话式采集用户画像
   → 保存到飞书多维表格

2. 用户画像采集完成
   → 行业研究员 spawn（输入：目标行业）
   → 能力评估师 spawn（输入：用户画像）
   → 并行执行

3. 两个报告完成
   → 规划师 spawn（输入：两个报告）
   → 生成转行计划

4. 所有报告完成
   → 整合成 PDF
   → 发送给用户
```

### 5.3 数据存储结构

**用户表**：
- user_id
- 姓名
- 邮箱
- 创建时间

**咨询记录表**：
- record_id
- user_id
- 目标行业
- 用户画像 JSON
- 行业调研报告链接
- 能力评估报告链接
- 转行计划链接
- 状态（进行中/已完成）

## 七、风险与应对

### 7.1 技术风险
- **风险**：AI 回复质量不稳定
- **应对**：建立质量检查机制，人工审核关键报告

### 7.2 商业风险
- **风险**：用户付费意愿低
- **应对**：提供高价值付费功能（真人咨询、内部数据）

### 7.3 竞争风险
- **风险**：大厂推出类似产品
- **应对**：
  - 专注垂直领域
  - 积累行业数据
  - 建立用户社区
