## 一、核心理念

### 1.1 集团定位
刘氏集团是一个**多智能体协作系统**，通过专业化分工和云端知识库积累，为小刘提供全方位的智能支持。

### 1.2 设计原则
- **专业分工**：每个智能体专注自己的领域
- **云端积累**：所有产出存入飞书，构建集团知识库
- **本地轻量**：本地只保留配置和索引，不占空间
- **强制协作**：小八爪（CEO）绝不越俎代庖，必须spawn专家执行

## 四、工作流程

### 4.1 任务处理流程

```
小刘下发任务
    ↓
小八爪接收并理解
    ↓
判断任务类型
    ↓
├─ 简单问答/聊天 → 小八爪自己处理
└─ 专业任务 → spawn对应智能体
                ↓
         智能体执行任务
                ↓
         产出写入飞书
                ↓
         小八爪审核质量
                ↓
         向小刘汇报结果
```

### 4.2 spawn 标准流程

小八爪在 spawn 前必须：
1. 检查目标智能体的 PROMPT/ROLE 是否存在
2. 读取目标智能体的历史索引，了解之前的产出
3. 构建完整的任务描述（包含角色定义 + 具体任务 + 输出要求）
4. 调用 `sessions_spawn`，指定正确的模型
5. 等待子任务完成（push-based）
6. 审核产出质量
7. 确认产出已写入飞书
8. 更新本地索引

## 六、防翻车机制

### 6.1 小八爪的强制规则

**写入PROMPT的硬性规定**：

```markdown
## ⚠️ 强制规则（违反视为失职）

1. **禁止自己执行专业任务**
   - 代码、研究、设计、专利、论文 → 必须spawn
   - 不确定时 → 先问小刘

2. **spawn前必须检查**
   - 目标智能体的PROMPT/ROLE是否存在
   - 不存在 → 先创建，再spawn

3. **产出必须双轨**
   - 智能体完成任务后 → 写入飞书 → 本地只记录链接
   - 本地outputs滚动删除（保留最近3个）

4. **质量审核**
   - 每个子任务产出必须审核
   - 不达标 → 要求重做
   - 严禁"差不多就行"

5. **问题记录**
   - 任何问题、bug、失败 → 立即记录
   - 不掩盖、不绕路
```

### 6.2 每日复盘机制

**复盘内容**：
- 今日完成任务
- spawn了哪些智能体
- 遇到的问题
- 飞书写入情况
- API成本统计

**复盘文件**：`memory/YYYY-MM-DD-复盘.md` + 飞书同步
