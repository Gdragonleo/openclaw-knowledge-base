# 每30分钟查询协作任务

## 任务说明

每30分钟自动查询协作中心的多维表格，检查是否有新任务需要执行。

## 执行时间

- **触发时间**：每30分钟（每小时的00分和30分）
- **Cron表达式**：`0,30 * * * *`

## 执行步骤

### 1. 查询多维表格

**多维表格信息**：
- App Token: `EHhIblp0va1NlvshBNHclHRynRb`
- Table ID: `tblXFQ67LNLRmmGU`

**查询条件**：
- 任务状态 = ⏳ 待处理
- 接收者 = 🐙 小八爪 OR 👥 双方

### 2. 处理任务

**任务类型处理**：

#### [BOT_TASK] 任务指令
- 按优先级排序（5→1）
- 逐个执行任务
- 执行后更新状态为"✅ 已完成"
- 记录完成时间

#### 💬 日常交流
- 查看内容
- 根据需要回复（通过多维表格或飞书私聊）
- 更新状态为"✅ 已完成"

#### 🔔 紧急通知
- 立即处理
- 高优先级（5）
- 完成后立即通知小刘

#### 📊 状态更新
- 记录状态信息
- 更新状态为"✅ 已完成"

### 3. 任务执行日志

**保存位置**：`outputs/小八爪/2026-03/团队协作/任务执行日志_YYYY-MM-DD.md`

**日志格式**：
```markdown

- 处理结果: 进行中...
```

### 4. 通知机制

**完成任务后**：
- 在多维表格中更新状态
- 记录完成时间和结果
- 如果是重要任务，私聊通知小刘

## 错误处理

### 问题1：无法访问多维表格

**症状**：API返回权限错误

**解决**：
- 检查App Token和Table ID是否正确
- 确认飞书应用权限配置
- 记录错误，下次重试

### 问题2：任务执行失败

**症状**：任务无法完成

**解决**：
- 更新状态为"❌ 已取消"
- 记录失败原因
- 通知小刘

### 减少API调用

- 每次查询只获取最近1小时的任务
- 避免重复查询同一个任务
- 使用本地缓存记录已处理的任务ID
