# 每日招标信息超详细报告任务

## 任务目标
每天自动抓取前一天发布的招标信息，筛选勘察/监测类项目，生成超详细报告并上传到飞书文档。

## 执行时间
每天凌晨 5:00 执行

## 任务步骤

### 第一步：数据抓取
访问多个公共资源交易中心网站，抓取前一天发布的招标公告。

**时间范围**：昨天（例如今天是3月13日，则抓取3月12日发布的项目）
**报告命名**：招标日报_YYYY-MM-DD（使用**数据日期**，即昨天的日期）

**目标网站（优先级排序）**：
1. ⭐ **重庆市公共资源交易中心**（https://www.cqggzy.com）- 优先级1
2. ⭐ **四川省公共资源交易中心**（http://ggzyjy.sc.gov.cn）- 优先级2
3. ⭐ **湖北省公共资源交易中心**（https://www.hbggzyfwpt.cn）- 优先级2
4. ⭐ **贵州省公共资源交易中心**（http://ggzy.guizhou.gov.cn）- 优先级2
5. ⭐ **云南省公共资源交易中心**（https://www.ynggzyxx.gov.cn）- 优先级3

**执行策略**：
- 使用 tender-browser-scraper 抓取数据
- 使用 whale 的7维度评分系统
- 按优先级顺序爬取
- 每个网站最多抓取50个项目
- 随机延迟3-8秒，避免触发反爬
- 遇到验证码暂停并飞书通知
- **评分维度**（whale系统）：
  1. 业务方向（0-96分）
  2. 地区匹配（68-95分）
  3. 金额匹配（66-90分）
  4. 客户关系（72-94分）
  5. 成功案例（65-92分）
  6. 时间安排（62-90分）
  7. 特殊需求（70-98分）

### 第二步：项目筛选
使用关键词筛选勘察/监测类项目：

**第一优先级**：
- 勘察、地质勘察、岩土工程、岩土勘察
- 测绘、测量、地形测绘、工程测量
- 监测、检测、观测、安全监测、变形监测
- 物探、地质勘探

**第二优先级**：
- 水文地质、环境地质
- 边坡、基坑、地基
- 土壤污染、环境修复

### 第三步：生成超详细报告（使用 whale 7维度评分）
为每个筛选出的项目生成专业分析：

1. **📋 基本信息**
   - 项目名称、金额、招标单位
   - 发布时间、截止时间
   - 地区、项目类型
   - 资质要求、原文链接
   - **推荐指数**（whale 7维度平均分）
   - **中标概率**（基于推荐指数）

2. **📊 7维度评分**（whale 系统）
   - 业务方向匹配度
   - 地区匹配度
   - 金额匹配度
   - 客户关系评估
   - 成功案例匹配
   - 时间安排评估
   - 特殊需求匹配

3. **🎯 投标建议**（whale 策略）
   - 跟进策略（积极跟进/标准跟进/谨慎报价）
   - 报价建议（基于历史数据）
   - 建议折扣（2-5%）
   - 中标概率预测

### 第四步：集成 whale 系统

**执行步骤**：
1. 运行爬虫脚本：`cd ~/.openclaw/workspace/skills/tender-browser-scraper/scripts && node scrape.js --site cqggzy --days 1 --max 50`
2. 运行集成脚本：`cd ~/.openclaw/workspace/skills/tender-browser-scraper && python3 integrate_with_whale.py`
3. 验证 whale 数据：`cat ~/Desktop/项目管理平台代码/whale/whale/data/crawled_projects.json | jq '.projects | length'`

**输出文件**：
- 📄 Markdown报告：`知识库/招标监控/招标日报_whale.md`
- 📊 whale数据：`~/Desktop/项目管理平台代码/whale/whale/data/crawled_projects.json`
- 🗄️ whale数据库：`~/Desktop/项目管理平台代码/whale/whale/data/tenders.db`（通过 sync_data.py 更新）

### 第五步：发送飞书通知

**⚠️ 重要**：必须使用 feishu_doc 工具，create 后必须 write！

**执行步骤**：
1. 使用 `feishu_doc(action=create, title="招标日报_YYYY-MM-DD")` 创建文档
2. 获取返回的 doc_token
3. **必须**：使用 `feishu_doc(action=write, doc_token=XXX, content="完整报告内容")` 写入内容
4. 验证返回的 blocks_added > 0

**文档位置**：
- 父文件夹：刘氏集团知识库 > 招标监控
- 文件名格式：`招标日报_YYYY-MM-DD.md`
- 例如：`招标日报_2026-03-09.md`

**文档结构**：

## 重要提示

1. **数据时间**：只抓取前一天发布的项目，避免重复
2. **去重机制**：检查项目是否已在之前的报告中出现
3. **文件命名**：使用日期作为文件名，方便查找
4. **飞书上传**：确保文档正确上传到指定文件夹
5. **失败重试**：如果抓取失败，自动重试3次
