# 每日工作学习汇报任务（当前版）

## 任务目标
每天晚上 23:00 汇总当天工作进展、触手增长、重要事项与项目状态，生成可信的私聊汇报发给小刘。

## 执行时间
每天晚上 23:00 执行

## 通知方式
- ✅ 私聊小刘
- ❌ 不发飞书群
- ❌ 不走旧版群通知流程

## 汇报内容

### 1. 刘氏集团工作进展
- 当天推进了哪些实际事项
- 是否有完成/暂停/待继续的项目

### 2. 触手增长进度
- 当前触手数量
- 当天是否新增技能或新的能力沉淀
- 是否有值得记录的成长节点

### 3. 其他重要事项
- 定时任务异常
- 协作仓库进展
- 小鲸鱼协作动态
- 需要小刘知晓的风险或提醒

### 4. 项目管理仪表板更新（若有）
- Git 仓库状态
- 关键项目进展
- 风险预警

## 建议信息源
- `知识库/小八爪/`
- `知识库/招标监控/`
- `memory/YYYY-MM-DD.md`
- `memory/错误日志.md`
- `~/.openclaw/cron/jobs.json`
- `agent-collaboration` 当前 issue 动态（如有新增）

## 验证逻辑（必须执行）

### 内容完整性验证
汇报中至少要包含以下 3 项中的 2 项：
- 当天工作进展
- 触手/成长信息
- 风险/提醒/重要事项

### 质量验证
- 不能只是空模板
- 不能只写“今日无事”除非确实无新增且明确说明原因
- 若引用项目进展，必须基于真实文件/仓库/任务状态

### 发送前自检
- 是否有一句清晰结论
- 是否提到最重要的 1-3 个变化
- 是否避免冗长流水账

## 草稿脚手架
生成汇报前，可先运行：
- `python3 /Users/danxiong/.openclaw/workspace/scripts/cron/generate_work_report_draft.py`

用于快速汇总当日记忆与 cron 状态，减少空模板风险。
草稿文件会写入：
- `知识库/小八爪/2026-03/工作汇报/草稿_YYYY-MM-DD.md`
状态文件会写入：
- `运行状态/cron-validation/`

## 失败处理
若无法生成有效汇报：
1. 明确说明缺少哪些信息源
2. 使用最小可信版本发送，不得伪造进展
3. 若任务链路异常，直接发告警而不是沉默
