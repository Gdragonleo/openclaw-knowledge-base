# 每日错误复盘任务配置

## 任务信息

**任务名称**：每日错误复盘
**触发时间**：每天早上 6:00
**任务类型**：定时任务 + 飞书通知

## 任务目标

通过每日复盘错误日志，识别高频错误类型，生成易错提醒，帮助小八爪避免重复犯错。

## 执行步骤

### 1. 读取错误日志
- 文件路径：`/Users/danxiong/.openclaw/workspace/memory/错误日志.md`
- 读取完整内容

### 2. 统计高频错误
- 统计范围：本周（最近7天）
- 筛选条件：出现次数 ≥ 2次
- 分类统计：错误类型、出现次数、错误编号

### 3. 生成今日提醒
- 输出文件：`/Users/danxiong/.openclaw/workspace/memory/今日提醒.md`
- 内容格式：
  - 日期和生成时间
  - 易错提醒（基于高频错误）
  - 本周错误统计
  - 今日任务状态

### 4. 发送飞书通知
- 目标群：行业咨询群
- chat_id：`oc_eab876d73f2ffb6f89697181a232f130`
- 内容：简洁的易错提醒摘要

## 通知格式

```

【最高频】错误类型 N次
→ 预防措施

【次高频】错误类型 M次
→ 预防措施

详细提醒已更新到 memory/今日提醒.md
```

## 错误分类标准

### 按类型分类
1. **子智能体失败**：spawn的子智能体执行失败
2. **配置遗漏**：缺少必要的配置或参数
3. **重复犯错**：犯过同样的错误
4. **权限问题**：API权限、文件权限等
5. **知识指导错误**：凭直觉给出错误指导

### 按严重程度分类
- 🔴 高：影响用户体验、任务失败
- 🟡 中：影响效率、需要修复
- 🟢 低：可忽略、已解决

## 维护规则

1. **每次犯错后**：
   - 立即记录到 `memory/错误日志.md`
   - 格式：错误描述 / 场景 / 原因 / 正确做法 / 状态 / 人物

2. **每周回顾**：
   - 统计本周错误总数
   - 识别高频错误类型
   - 更新预防措施

3. **持续优化**：
   - 不再犯同样错误
   - 形成规则和习惯
   - 更新SOUL.md、AGENTS.md等文件
