# feishu-doc-upload

**一次调用，自动完成飞书文档创建和上传**

## 功能说明

解决"创建空白文档"问题：一次性完成：读取本地文件 → 创建文档 → 写入内容 → 验证 → 返回结果

**避免重复操作，提高效率**

## 使用方式

**基础用法**：
```
上传文件 `/path/to/file.md` 到 飞书文档"学习笔记"
```

**批量上传**：
```
上传目录 `/path/to/folder` 到 飞书文档
```

## 技术实现
**核心流程**：
1. **读取文件**：使用 `read` 工具读取本地MD文件
2. **创建文档**：使用 `feishu_doc create` 创建文档
3. **写入内容**：使用 `feishu_doc write` 写入内容
4. **验证结果**：使用 `feishu_doc read` 验证文档非空
5. **返回结果**：返回文档URL和状态

**错误处理**：
- 如果create失败，记录错误并返回
- 如果write失败,记录错误并重试
- 如果验证失败,记录错误并警告用户

## 使用示例

**示例1：上传单个文件**
```
用户: 上传学习笔记.md到飞书
助手: 
  - 读取文件内容
  - 创建文档"学习笔记"
  - 写入完整内容
  - 验证成功
  - 返回: ✅ 文档已上传 - https://feishu.cn/docx/xxx
```

**示例2:批量上传目录**
```
用户: 上传所有历史报告到飞书
助手:
  - 列出目录下所有.md文件
  - 逐个上传
  - 返回上传结果汇总
```

## 重要提醒
⚠️ **必须遵循的规则**：
1. **create后必须write**：API设计如此，不能跳过write步骤
2. **必须验证**：每次上传后验证文档有内容
3. **失败重试**：如果失败,最多重试2次
4. **错误记录**：失败时记录到错误日志,学习改进

## 技能参数
- **支持的文件类型**： .md, .txt
- **最大文件大小**： 建议 < 5MB
- **超时设置**： 300秒（5分钟)
- **重试次数**： 最多2次

## 错误记录
所有错误都会记录到 `memory/错误日志.md`
