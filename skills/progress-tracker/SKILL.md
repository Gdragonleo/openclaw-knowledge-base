# progress-tracker

**让所有操作进度可见，解决"不知道卡了还是在工作"的问题**

## 功能说明

这个skill强制我在处理复杂任务时：
1. ✅ **先列出步骤**（让用户知道我要做什么）
2. ✅ **逐步执行**（一步一步来）
3. ✅ **实时汇报**（每完成一步都告诉用户）

---

## 使用场景

**必须使用此skill的任务**：
- 🔍 分析多个文件或图片
- 🔧 排查技术问题
- 📚 处理大量数据
- 🔄 复杂的配置修改
- 📊 生成报告或总结

**不需要使用的场景**：
- 简单的读取单个文件
- 简单的消息回复
- 已有明确流程的日常任务

**示例**：

## 进度标签说明

| 标签 | 含义 |
|------|------|
| 📋 | 任务步骤列表 |
| 🔄 | 正在执行中 |
| ✅ | 已完成 |
| ❌ | 失败/出错 |
| ⏳ | 等待资源 |
| ⚠️ | 需要注意 |

---

## 强制规则

**遇到以下情况，必须使用progress-tracker**：

1. **任务超过3个步骤**
2. **涉及文件操作**
3. **需要网络请求**
4. **处理图片/视频**
5. **技术问题排查**
6. **用户明确要求看进度**

**不使用的情况**：
- 单步简单操作
- 用户说"不用汇报"
- 紧急情况需要快速响应

## 技术实现

**触发方式**：
- 自动检测：任务步骤>3 → 自动启用
- 手动触发：用户说"显示进度"、"让我看到过程"

**执行逻辑**：
1. 分析任务复杂度
2. 分解成具体步骤
3. 估算每步耗时
4. 逐步执行并汇报
5. 完成后总结
