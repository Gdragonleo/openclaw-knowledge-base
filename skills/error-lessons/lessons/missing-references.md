# 教训8：调研内容未附上出处链接

**错误类型**：调研报告
**严重程度**：⚠️ 中等（影响可信度）
**创建时间**：2026-03-13 08:58

---

## 问题场景

完成调研后，提供方案和建议，但**没有引用任何外部来源**，只基于自身经验总结。

---

## 错误现象

```
用户：调研下怎么把PDF文件或者飞书文档分享给微信

我的回复：
- 方法1：直接分享链接
- 方法2：导出为PDF
- ...
（没有引用任何来源）❌

用户反馈：
> "你在调研任何内容的时候，最后附上出处的链接"
```

---

## 根本原因

1. ❌ **没有使用搜索工具**：仅凭经验总结
2. ❌ **缺少验证**：没有查找官方文档或权威来源
3. ❌ **可信度不足**：用户无法验证建议的正确性
4. ❌ **缺少溯源**：无法追踪信息来源

---

## 影响范围

- ⚠️ 建议可信度降低
- ⚠️ 用户无法验证正确性
- ⚠️ 可能传播过时或错误信息
- ⚠️ 缺少专业度

---

## 解决方案

### **调研流程（必须执行）**：

```
1. ✅ 使用搜索工具查找权威来源
   - web_search（如果可用）
   - crawl4ai-skill（搜索+爬取）
   - 官方文档、技术博客、问答网站

2. ✅ 记录所有参考来源
   - 链接
   - 标题
   - 发布时间（如果可能）

3. ✅ 在报告末尾附上参考文献
   格式：
   ---
   ## 参考资料
   1. [标题](链接) - 来源/发布时间
   2. [标题](链接) - 来源/发布时间

4. ✅ 标注建议的来源
   - "根据飞书官方文档..."
   - "根据微信开发者文档..."
   - "根据经验总结..."（如果确实没有外部来源）
```

---

## 预防规则

### **调研任务开始时**：
```
[ ] 使用搜索工具查找信息
[ ] 记录所有有用的链接
[ ] 优先使用官方文档
[ ] 验证信息的时效性
```

### **调研报告输出时**：
```
[ ] 末尾添加"参考资料"章节
[ ] 列出所有参考链接
[ ] 标注每个建议的来源
[ ] 如果是经验总结，明确说明
```
