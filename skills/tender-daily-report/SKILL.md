# 招标日报自动生成工具

一条命令完成：**爬取 → 硬筛选 → 7维度评分 → 生成报告 → 按日期归档 → 验证产物**

## 🚀 快速使用

```bash
# 完整流水线（爬取+评分+报告）
bash scripts/run_pipeline.sh

# 仅评分+报告（已有爬取数据时）
python3 scripts/score_and_report.py

# 验证产物是否生成
python3 scripts/verify_outputs.py
```

## ⚙️ 配置

所有配置在 `config/` 目录下，修改后立即生效。

### 1. 站点配置 `config/sites.json`
配置要爬取的公共资源交易中心。

### 2. 关键词配置 `config/keywords.json`
- **primary**: 高优先级关键词（勘察、测绘、监测等）
- **secondary**: 次要关键词（水文地质、边坡等）
- **hard_exclude**: 硬排除词（监理、施工、EPC等标题含这些词的项目直接过滤）

### 3. 公司画像 `config/company_profile.json`
7维度评分依据，包含：
- 偏好业务类型、偏好地区、金额范围
- 长期客户、成功案例关键词
- 当前产能、特殊关注点

### 4. 运行配置 `config/settings.json`
- 爬取天数范围（默认30天）
- 最大抓取条数（默认12条）
- 输出目录（默认 `知识库/招标监控/`）
- 是否自动通知

## 📊 输出产物

每次运行后，在输出目录按日期生成：

```
知识库/招标监控/
├── 2026-03-20/                    # 按日期归档
│   ├── 招标日报_2026-03-20.md     # Markdown日报
│   ├── 招标日报_2026-03-20.html   # HTML分享页（如有）
│   ├── 招标日报_2026-03-20_projects.json   # 项目详情
│   └── 招标日报_2026-03-20_summary.json   # 统计汇总
├── _latest/                       # 最新版快捷入口（符号链接）
└── 历史报告/                      # 自动归档
```

## 📋 流水线步骤

```
[1/4] 爬取招标公告
       ↓ (失败则降级)
[2/4] quick_report 降级方案
       ↓
[3/4] 7维度评分 + 硬筛选
       ↓
[4/4] 生成报告 + 按日期归档 + 验证
```

### 7维度评分说明

| 维度 | 分值范围 | 说明 |
|------|---------|------|
| 业务方向 | 0-96 | 项目类型与公司业务匹配度 |
| 地区匹配 | 68-95 | 项目地区是否在偏好范围 |
| 金额匹配 | 66-90 | 项目金额是否在预算范围 |
| 客户关系 | 72-94 | 是否为长期客户 |
| 成功案例 | 65-92 | 关键词匹配历史成功案例 |
| 时间安排 | 62-90 | 截止时间是否充裕 |
| 特殊需求 | 70-98 | 是否符合特殊关注点 |

**推荐指数** = 7维度平均分
**中标概率** = 基于推荐指数计算

```bash
sessions_spawn --task "执行招标日报流水线: bash skills/tender-daily-report/scripts/run_pipeline.sh，完成后验证产物并发送私聊通知给小刘" --model zai/glm-5
```

## ⚠️ 注意事项

1. **合规使用** - 仅抓取公开招标信息，遵守robots.txt
2. **真实数据** - 禁止使用样例/虚构数据填充报告
3. **原链接保留** - 每个项目必须保留原文链接可回源
4. **硬筛选不可跳过** - 监理/施工/EPC 标题类项目必须过滤

## 📄 依赖

- Python 3 >= 3.9（评分+报告）
