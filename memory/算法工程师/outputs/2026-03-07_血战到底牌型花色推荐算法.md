# 血战到底牌型花色推荐算法设计文档

## 1. 概述

本文档描述“麻将算命微信小程序”的核心算法需求。产品主打四川血战到底玩法，系统需根据用户输入的生辰八字/星座/日期等信息，推荐用户近期适合的牌型（如对对胡、清一色、七对等）以及花色倾向（万、筒、条）。

## 2. 输入定义

### 2.1 用户输入
- `birth_datetime`：生辰八字（公历年月日时分，可转换为农历八字）
- `zodiac_sign`：星座（12星座之一）
- `query_date`：查询日期（公历，用于计算节气、农历等）
- 可选：`gender`（性别，部分流派考虑）

### 2.2 预处理
- 将公历生辰转换为农历八字（年柱、月柱、日柱、时柱）
- 提取八字中的天干地支
- 计算八字五行强度（金、木、水、火、土）
- 计算生肖
- 查询日期转换为农历日期、节气信息

## 3. 输出定义

### 3.1 牌型推荐
- `recommended_pattern`：推荐牌型（字符串枚举）
- `pattern_score`：牌型匹配度（0-100分）
- `alternative_patterns`：备选牌型列表（按分数降序）

**常见血战到底牌型枚举**：
- 平胡（基本胡牌）
- 对对胡（碰碰胡）
- 清一色（同一花色）
- 混一色（同一花色+字牌）
- 七对（七个对子）
- 清对（清一色+对对胡）
- 将对（只含2、5、8的对对胡）
- 门清（无吃碰明杠）
- 杠上花（杠后胡牌）
- 杠上炮（杠后点炮）
- 抢杠胡
- 海底捞月（最后一张牌自摸）
- 血战到底（流局时未胡玩家继续）

### 3.2 花色倾向
- `recommended_suit`：推荐花色（万、筒、条）
- `suit_score`：花色缘分度（0-100分）
- `suit_breakdown`：各花色详细得分

## 4. 核心算法逻辑

### 4.1 五行计算模块

#### 4.1.1 八字五行强度
根据生辰八字计算五行强度矩阵：

1. 年柱、月柱、日柱、时柱各对应一个天干和一个地支
2. 每个天干地支对应五行属性（例如：甲乙属木，丙丁属火，戊己属土，庚辛属金，壬癸属水；子亥属水，寅卯属木，巳午属火，申酉属金，辰戌丑未属土）
3. 统计各五行出现的次数，并考虑“藏干”影响（地支中隐藏的天干）
4. 最终得到五维向量 `W = [金, 木, 水, 火, 土]`，归一化到[0,1]区间

#### 4.1.2 五行平衡指数
计算五行平衡度：
```
balance_index = 1 - (std(W) / max_std)
```
其中`std(W)`为五行强度的标准差，`max_std`为理论最大标准差（极端不平衡情况）。

平衡指数高表示五行均衡，低表示某一行特别强或特别弱。

### 4.2 牌型推荐算法

#### 4.2.1 牌型特征矩阵
为每种牌型定义特征向量，包含以下维度：
- `aggressiveness`：进攻性（0-1）
- `complexity`：复杂度/难度（0-1）
- `luck_factor`：运气成分（0-1）
- `risk_tolerance`：风险容忍度（0-1）
- `patience`：耐心要求（0-1）

示例特征（需产品经理与麻将专家确认）：
| 牌型 | 进攻性 | 复杂度 | 运气成分 | 风险容忍度 | 耐心 |
|------|--------|--------|----------|------------|------|
| 平胡 | 0.3 | 0.2 | 0.5 | 0.8 | 0.6 |
| 对对胡 | 0.7 | 0.5 | 0.4 | 0.6 | 0.4 |
| 清一色 | 0.8 | 0.8 | 0.7 | 0.3 | 0.9 |
| 七对 | 0.6 | 0.7 | 0.8 | 0.5 | 0.8 |
| ... | ... | ... | ... | ... | ... |

#### 4.2.2 用户特征向量
从用户输入推导用户特征向量：

1. **五行特征**：
   - 金强 → 进攻性+0.3，耐心-0.2
   - 木强 → 复杂度+0.2，风险容忍度+0.1
   - 水强 → 运气成分+0.3，耐心+0.2
   - 火强 → 进攻性+0.4，风险容忍度+0.3
   - 土强 → 耐心+0.3，运气成分-0.1

2. **星座特征**（简化映射）：
   - 火象星座（白羊、狮子、射手）：进攻性+0.2，风险容忍度+0.2
   - 土象星座（金牛、处女、摩羯）：耐心+0.3，运气成分-0.1
   - 风象星座（双子、天秤、水瓶）：复杂度+0.3，耐心-0.1
   - 水象星座（巨蟹、天蝎、双鱼）：运气成分+0.3，进攻性-0.1

3. **日期特征**：
   - 节气：立春、立夏、立秋、立冬当日 → 对应五行属性增强
   - 农历初一/十五：运气成分波动
   - 本命年/冲太岁：风险容忍度调整

4. **平衡指数影响**：
   - 平衡指数高 → 复杂度偏好降低，平胡倾向增加
   - 平衡指数低 → 极端牌型（清一色、七对）倾向增加

最终得到用户特征向量 `U = [aggressiveness, complexity, luck_factor, risk_tolerance, patience]`，归一化到[0,1]。

#### 4.2.3 匹配度计算
计算用户特征向量U与牌型特征向量P的余弦相似度：
```
similarity = (U · P) / (||U|| * ||P||)
```
将相似度映射到0-100分。

#### 4.2.4 牌型评分修正
考虑以下修正因子：
- **五行平衡修正**：五行极不平衡时，对对胡、清一色等极端牌型加分
- **节气修正**：当前节气对应的五行相关牌型加分
- **生肖修正**：用户生肖与当年太岁关系影响风险偏好

最终牌型得分：
```
final_score = similarity * 100 * (1 + correction_factor)
```

### 4.3 花色倾向算法

#### 4.3.1 五行与花色映射
建立五行与麻将花色的对应关系：

| 五行 | 对应花色 | 理由 |
|------|----------|------|
| 金   | 筒（饼） | 圆形似金元，金属质感 |
| 木   | 条（索） | 条状似竹木，绿色象征 |
| 水   | 万       | 万源归水，流动不息 |
| 火   | （无直接对应） | 火通常对应"红中"等字牌 |
| 土   | （无直接对应） | 土可视为基础，所有花色皆含 |

**注**：这是基于传统文化和麻将象征的合理映射，可根据专家意见调整。

#### 4.3.2 花色得分计算

1. **基础得分**：
   ```
   花色得分 = 对应五行强度 * 100
   ```
   例如：用户木行强度0.8 → 条花色基础得分80

2. **天干地支修正**：
   - 日柱天干为甲乙（木） → 条花色+20
   - 日柱天干为庚辛（金） → 筒花色+20
   - 日柱天干为壬癸（水） → 万花色+20
   - 地支含寅卯（木） → 条花色+10
   - 地支含申酉（金） → 筒花色+10
   - 地支含亥子（水） → 万花色+10

3. **节气修正**：
   - 春季（木旺） → 条花色+15
   - 夏季（火旺） → 筒花色（火生土，土生金）间接+10
   - 秋季（金旺） → 筒花色+15
   - 冬季（水旺） → 万花色+15

4. **生肖修正**：
   - 生肖虎、兔（木） → 条花色+10
   - 生肖猴、鸡（金） → 筒花色+10
   - 生肖鼠、猪（水） → 万花色+10

#### 4.3.3 最终花色推荐
计算万、筒、条三门的最终得分，归一化到0-100分，推荐最高分花色。

### 4.4 综合推荐逻辑

1. 优先推荐牌型与花色匹配的组合（如推荐清一色+万花色，则用户可尝试做清一色万子）
2. 若牌型为"混一色"，则花色推荐权重降低
3. 若牌型为"平胡"，则花色倾向仅供参考

## 5. 边界条件处理

### 5.1 输入缺失处理
- 生辰八字缺失：使用默认五行均衡分布，仅依赖星座和日期
- 星座缺失：根据生辰月份推算太阳星座
- 所有输入缺失：返回默认推荐（平胡 + 均衡花色）

### 5.2 分数平局处理
当多个牌型/花色分数接近时（差值<5分）：
- 牌型：推荐复杂度较低的（用户体验友好）
- 花色：推荐当前节气对应的五行花色

### 5.3 不合理组合过滤
- 七对 + 特定花色：七对不强调花色，花色推荐仅供参考
- 门清 + 杠上花：逻辑冲突，应避免同时推荐

## 6. 算法复杂度分析

- 五行计算：O(1)，固定8个天干地支
- 牌型匹配：O(n)，n为牌型数量（约12种）
- 花色计算：O(1)，3种花色
- 总体复杂度：O(1)，适合微信小程序实时计算

## 8. 扩展性设计

### 8.1 机器学习演进
预留特征接口，未来可收集用户实际打牌胜负数据，训练推荐模型：
- 特征：用户输入 + 推荐牌型/花色 + 实际游戏结果
- 标签：是否胡牌、胡牌番数
- 模型：逻辑回归/XGBoost，定期更新推荐策略

### 8.2 A/B测试支持
算法支持多套参数配置，便于进行A/B测试验证推荐效果。

## 9. 实现建议

### 9.1 前端实现
- 微信小程序端：仅做简单计算，复杂八字转换调用后端API
- 缓存用户最近计算结果，减少重复计算

### 9.2 后端实现
- 提供RESTful API：`POST /api/mahjong/fortune`
- 请求参数：`{birth_datetime, zodiac_sign, query_date}`
- 返回JSON：`{recommended_pattern, pattern_score, recommended_suit, suit_score, explanation}`

### 9.3 数据更新
- 节气、农历数据每年更新一次
- 牌型特征矩阵可后台配置，热更新

## 10. 验证方法

1. **专家验证**：邀请麻将高手评估推荐合理性
2. **历史数据验证**：收集历史牌局，分析推荐牌型与实际胡牌牌型的相关性
3. **用户反馈**：小程序内设置"推荐是否准确"反馈入口
