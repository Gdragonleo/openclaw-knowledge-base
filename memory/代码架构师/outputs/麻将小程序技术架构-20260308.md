# 麻将算命微信小程序技术架构方案

## 1. 技术栈选型

### 前端框架
- **微信小程序原生框架 + WXML/WXSS/JavaScript**
  - 理由：官方支持最好，性能最优，无需额外学习成本
  - 使用小程序云开发能力，实现快速开发部署

### 云开发平台
- **微信小程序云开发**
  - 云函数：用于业务逻辑处理
  - 云数据库：存储用户数据、算命记录等
  - 云存储：存储图片、配置文件等
  - 理由：免服务器运维，快速上线，与小程序生态无缝集成

### 数据库
- **微信云数据库（基于MongoDB）**
  - 理由：NoSQL数据库，灵活的数据结构，适合算命业务的数据存储
  - 支持实时数据同步，适合用户交互场景

### 第三方服务
- **腾讯位置服务**：用于获取用户地理位置（可选）
- **微信支付**：用于付费算命功能（可选）
- **腾讯云AI能力**：用于智能推荐（可选）

## 3. 核心模块划分

### 3.1 排盘模块
- **功能**：根据用户输入信息生成麻将牌阵
- **输入**：用户生日、性别、当前时间、问题类型
- **输出**：14张麻将牌组成的牌阵
- **算法**：基于传统麻将算命算法，结合随机数生成

### 3.2 结果展示模块
- **功能**：解析牌阵，生成算命结果
- **输入**：麻将牌阵
- **输出**：运势解读、建议、注意事项
- **展示形式**：文字解读 + 可视化图表 + 分享功能

### 3.3 用户管理模块
- **功能**：用户注册、登录、历史记录管理
- **核心能力**：
  - 微信授权登录
  - 用户信息存储
  - 算命历史记录
  - 收藏功能

### 3.4 数据统计模块
- **功能**：统计用户行为、热门算命类型
- **用途**：优化算法、个性化推荐

## 4. API接口设计

### 4.1 用户相关接口

#### 4.1.1 用户登录
- **接口**：`/user/login`
- **方法**：POST
- **入参**：
  ```json
  {
    "code": "微信登录code"
  }
  ```
- **出参**：
  ```json
  {
    "openid": "用户唯一标识",
    "session_key": "会话密钥",
    "userInfo": {
      "nickName": "昵称",
      "avatarUrl": "头像"
    }
  }
  ```

#### 4.1.2 获取用户信息
- **接口**：`/user/info`
- **方法**：GET
- **入参**：无（通过header token验证）
- **出参**：
  ```json
  {
    "userId": "用户ID",
    "nickName": "昵称",
    "avatarUrl": "头像",
    "fortuneCount": "算命次数",
    "lastFortuneTime": "最后算命时间"
  }
  ```

### 4.2 算命相关接口

#### 4.2.1 生成牌阵
- **接口**：`/fortune/generate`
- **方法**：POST
- **入参**：
  ```json
  {
    "birthday": "1990-01-01",
    "gender": "male/female",
    "questionType": "career/love/health",
    "timestamp": "当前时间戳"
  }
  ```
- **出参**：
  ```json
  {
    "fortuneId": "算命记录ID",
    "mahjongArray": ["一万", "二万", ..., "白板"],
    "generatedTime": "生成时间"
  }
  ```

#### 4.2.2 获取算命结果
- **接口**：`/fortune/result`
- **方法**：POST
- **入参**：
  ```json
  {
    "fortuneId": "算命记录ID"
  }
  ```
- **出参**：
  ```json
  {
    "fortuneId": "算命记录ID",
    "overallFortune": "整体运势",
    "details": {
      "career": "事业运势",
      "love": "感情运势",
      "health": "健康运势",
      "wealth": "财运"
    },
    "advice": "建议",
    "luckyColor": "幸运颜色",
    "luckyNumber": "幸运数字"
  }
  ```

#### 4.2.3 获取历史记录
- **接口**：`/fortune/history`
- **方法**：GET
- **入参**：
  ```json
  {
    "page": 1,
    "pageSize": 10
  }
  ```
- **出参**：
  ```json
  {
    "list": [
      {
        "fortuneId": "记录ID",
        "questionType": "问题类型",
        "resultSummary": "结果摘要",
        "createTime": "创建时间"
      }
    ],
    "total": 100
  }
  ```

### 4.3 支付相关接口（可选）

#### 4.3.1 创建支付订单
- **接口**：`/payment/create`
- **方法**：POST
- **入参**：
  ```json
  {
    "productId": "产品ID",
    "amount": 9.9
  }
  ```
- **出参**：
  ```json
  {
    "orderId": "订单ID",
    "paymentParams": "微信支付参数"
  }
  ```

## 6. 性能优化方案

### 6.1 前端优化
- 使用小程序分包加载，减少首次加载时间
- 图片资源使用WebP格式，压缩体积
- 实现虚拟列表，优化长列表渲染

### 6.2 后端优化
- 云函数使用缓存机制，减少重复计算
- 数据库查询添加索引，优化查询性能
- 使用CDN加速静态资源访问

### 6.3 安全考虑
- 用户敏感信息加密存储
- API接口添加防刷机制
- 输入参数校验和过滤
