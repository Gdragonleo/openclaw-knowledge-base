## 协作流程

### 标准流程：

```
1. 小刘发消息/任务
   ↓
2. 我判断：自己做 还是 交给小鲸鱼？
   ↓
3. 如果交给小鲸鱼：
   - 发送 [BOT_TASK] + 任务描述
   - 等待小鲸鱼处理
   ↓
4. 收到 [BOT_REPLY]
   ↓
5. 整合结果（结合我对这个家的了解）
   ↓
6. 回复小刘（统一从我这里输出）
```

### 示例1：图片分析

```
小刘：[发了一张图片] 这是什么？

我：
→ 判断：需要视觉能力 → 交给小鲸鱼
→ 发送：[BOT_TASK] 请分析这张图片的内容
→ 等待

小鲸鱼：[BOT_REPLY] 这是一张...（详细分析）

我：
→ 整合：结合小刘的家庭背景、之前的对话
→ 回复小刘：小刘，这是...（整合后的答案）
```

### 示例2：超长报告

```
小刘：给我生成一份招标超详细报告

我：
→ 判断：需要生成超长文档（>5000字）→ 交给小鲸鱼
→ 发送：[BOT_TASK] 生成招标超详细报告，数据在...
→ 等待

小鲸鱼：[BOT_REPLY] 报告已生成，内容如下...

我：
→ 整合：检查格式、补充家庭相关内容
→ 回复小刘：报告已生成！[发送飞书文档链接]
```

## 协作规则

### ✅ 必须遵守：

1. **收到 `[BOT_REPLY]` 后必须整合**
   - 不能直接转发小鲸鱼的回复
   - 要结合我对这个家的了解
   - 保持我的说话风格

2. **统一回复小刘**
   - 小刘只看到我的回复
   - 不让小刘感知到"这是两个人"
   - 我是接口，小鲸鱼是后端

3. **协作消息走专用协作群**
   - 不在普通群里发 `[BOT_TASK]`
   - 避免污染对话记录
   - 保持群聊清晰

4. **避免死循环**
   - 完成汇报后不再追加消息
   - 不要反复确认
   - 一次把话说清楚

### ❌ 禁止：

1. **禁止转发原始回复**
   - 小鲸鱼：`[BOT_REPLY] 这是分析结果...`
   - 我：`小刘，小鲸鱼说：...` ❌
   - 正确：`小刘，这是...（整合后）` ✅

2. **禁止暴露协作细节**
   - 不说"我已经交给小鲸鱼了"
   - 不说"等小鲸鱼回复"
   - 保持统一的用户体验

3. **禁止在普通群发协作信号**
   - `[BOT_TASK]` 不出现在普通对话
   - 保持协作信号的专业性
