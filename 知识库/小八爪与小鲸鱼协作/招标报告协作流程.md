# 招标报告协作流程（小八爪 + 小鲸鱼）

---

## 📋 协作方案

**方案类型**：专项协作（小八爪收集 + 小鲸鱼深度分析）

**适用场景**：每日招标报告（每天9:00）

---

## 🤝 协作流程

### 阶段1：小八爪基础分析（9:00-10:00）

**执行者**：🐙 小八爪

**任务内容**：
1. **数据收集**
   - 爬取5个公共资源交易中心网站
   - 重庆：https://www.cqggzy.com
   - 四川：http://ggzyjy.sc.gov.cn
   - 湖北：https://www.hbggzyfwpt.cn
   - 贵州：http://ggzy.guizhou.gov.cn
   - 云南：https://www.ynggzyxx.gov.cn

2. **项目筛选**
   - 关键词：勘察、监测、测绘、检测、地质、岩土
   - 时间范围：昨天发布的招标公告
   - 数据量：每个网站最多50个项目

3. **7维度基础分析**
   - 📋 基本信息：项目名称、金额、招标单位、发布时间、截止时间
   - 📄 项目详情：批准文件、建设内容、项目规模
   - 💰 市场机会：政策支持、市场需求、推荐指数（1-5星）
   - ⚔️ 竞争分析：竞争程度（1-5星）
   - 🎯 投标建议：投标策略、准备时间
   - ⚠️ 风险评估：时间风险、业绩风险
   - 🔗 原文链接：可点击访问的完整URL

4. **识别重点项目**
   - 筛选标准：推荐指数 ≥ 4星
   - 输出：重点项目清单（项目名称、推荐理由、分析要点）

5. **保存基础报告**
   - 路径：知识库/招标监控/招标日报_YYYY-MM-DD.md
   - 仓库：https://gitee.com/whaleandcollab/collab-knowledge-base

6. **触发小鲸鱼协作**
   - 创建Gitee Issue
   - 标签：`tender-analysis`
   - 标题：`【招标分析】招标日报_YYYY-MM-DD - 重点项目深度分析`
   - 内容：
     ```markdown
     # 重点任务清单

     **报告日期**: YYYY-MM-DD
     **项目数量**: X个
     **重点项目**: Y个

     ---

     ## 重点任务1: [项目名称]
     - **推荐指数**: ⭐⭐⭐⭐⭐
     - **推荐理由**: [理由]
     - **分析要点**:
       1. 政策解读
       2. 竞争对手调研
       3. 市场机会评估
       4. 投标策略制定
       5. 风险评估

     ---

     ## 重点任务2: [项目名称]
     ...
     ```

7. **私聊通知小刘**
   - 内容：
     ```
     📊 每日招标报告（基础版）已完成！

     **日期**: YYYY-MM-DD
     **项目总数**: X个
     **重点项目**: Y个（已发给小鲸鱼深度分析）

     **基础报告**: [Gitee链接]
     **深度分析**: 预计12:00完成

     ——小八爪 🐙
     [时间]
     ```

### 阶段2：小鲸鱼深度分析（10:00-12:00）

**执行者**：🐋 小鲸鱼

**触发条件**：
- 自动：小鲸鱼每20分钟查询Gitee Issues
- 标签：`tender-analysis`

**任务内容**：
1. **查询Gitee Issue**
   - 读取小八爪创建的重点任务清单
   - 提取分析要点

2. **深度分析（每个重点项目）**
   - 🔍 **政策解读**
     - 地方政策支持
     - 行业发展趋势
     - 相关法规要求

   - 🏢 **竞争对手调研**
     - 主要竞争对手
     - 竞争对手优势劣势
     - 市场份额分析

   - 💰 **市场机会评估**
     - 市场需求分析
     - 投资回报预估
     - 业务拓展机会

   - 🎯 **投标策略制定**
     - 报价策略
     - 技术方案重点
     - 团队配置建议
     - 成功概率预测

   - ⚠️ **风险评估**
     - 时间风险
     - 业绩风险
     - 技术风险
     - 资金风险
     - 竞争风险

   - 📸 **图片分析**（如有需要）
     - 项目地理位置
     - 现场照片
     - 相关资料图片

3. **保存深度分析**
   - 路径：知识库/招标监控/深度分析_YYYY-MM-DD.md
   - 仓库：https://gitee.com/whaleandcollab/collab-knowledge-base

4. **更新Gitee Issue**
   - 状态：✅ 已完成
   - 评论：深度分析已完成，链接：[Gitee链接]

5. **私聊通知小刘**
   - 内容：
     ```
     🔍 招标报告深度分析已完成！

     **日期**: YYYY-MM-DD
     **分析项目**: X个重点项目
     **深度报告**: [Gitee链接]

     **核心发现**:
     1. [发现1]
     2. [发现2]
     3. [发现3]

     ——小鲸鱼 🐋
     [时间]
     ```

## ⚠️ 容错机制（防幺蛾子）

### 情况1：小鲸鱼未响应（10:30还没开始）

**触发条件**：
- 10:30查询Gitee Issue，状态仍是pending
- 小鲸鱼未在30分钟内响应

**处理方案**：
- 🐙 小八爪自动接管深度分析
- 独立完成所有重点项目分析
- 保证报告完整性
- 私聊通知小刘：小鲸鱼未响应，小八爪已独立完成

---

### 情况2：Gitee Issue创建失败

**触发条件**：
- 创建Issue时网络错误
- Gitee API调用失败

**处理方案**：
- 降级方案1：私聊发给小鲸鱼（通过飞书）
- 降级方案2：小八爪独立完成深度分析
- 保证报告完整性

---

### 情况3：爬取失败

**触发条件**：
- 网站反爬机制
- 网络连接超时
- 验证码拦截

**处理方案**：
- 重试3次（间隔10分钟）
- 失败后私聊通知小刘
- 部分成功也生成报告

---

### 情况4：数据量过大（>100个项目）

**触发条件**：
- 单日项目数量超过100个

**处理方案**：
- 只分析TOP 30重点项目
- 其余项目列表展示（基本信息）
- 私聊通知小刘：数据量大，已筛选重点
