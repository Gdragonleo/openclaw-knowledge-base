# 表单排序修改方案

**问题**：新增数据排在最后，需要改成排在最前  
**原因**：排序配置为升序（orderAsc: true）  
**解决**：改为倒序（orderAsc: false）

---

根据CSV配置，以下表单需要检查：

### 1. 商机信息列表（FORM_ID: 1935976031257104384）

**当前配置**：
```json
"orderColumn": "",
"orderAsc": true
```

**修改为**：
```json
"orderColumn": "CREATED_TIME",
"orderAsc": false
```

## 🔧 修改方法

### 方法1：通过管理后台

1. 登录你的平台管理后台
2. 找到表单配置
3. 修改列表组件的排序设置
4. 保存配置

### 方法2：直接修改数据库

```sql
-- 查找需要修改的表单配置
SELECT FORM_ID, FORM_NAME, WIDGET_JSON 
FROM ZZ_ONLINE_FORM 
WHERE FORM_CODE = 'list' OR FORM_CODE = 'sjgjlist';

-- 更新配置（示例）
UPDATE ZZ_ONLINE_FORM 
SET WIDGET_JSON = REPLACE(
  REPLACE(WIDGET_JSON, '"orderAsc":true', '"orderAsc":false'),
  '"orderColumn":""', '"orderColumn":"CREATED_TIME"'
)
WHERE FORM_ID IN ('1935976031257104384', '1937314401409306624');
```

### 方法3：修改CSV后导入

1. 在CSV文件中找到对应的行
2. 修改 `WIDGET_JSON` 列中的配置：
   - `"orderAsc":true` → `"orderAsc":false`
   - `"orderColumn":""` → `"orderColumn":"CREATED_TIME"`
3. 保存CSV
4. 导入到系统

## ⚠️ 注意事项

1. **备份数据**：修改前先备份表单配置
2. **测试验证**：修改后在测试环境验证
3. **刷新缓存**：修改后可能需要清理缓存
4. **影响范围**：只影响列表显示顺序，不影响数据本身

---

## 📊 排序字段说明

**常用排序列**：
- `CREATED_TIME` - 创建时间（推荐）
- `UPDATE_TIME` - 更新时间
- `BILLDATA` - 单据日期
- `PK_xxx` - 主键（通常按插入顺序）

**排序方向**：
- `orderAsc: true` - 升序（旧数据在前）
- `orderAsc: false` - 倒序（新数据在前）
