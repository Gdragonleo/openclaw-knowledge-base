## 一、常见页面结构变化模式

### 1.1 公共资源交易中心页面的典型变化

| 变化类型 | 频率 | 影响 |
|---------|------|------|
| CSS类名变更 | 高（每次改版） | 选择器失效 |
| DOM层级调整 | 中（功能迭代） | XPath失效 |
| 新增包裹div | 高（样式调整） | 级联选择器失效 |
| 表格→div列表 | 低（大改版） | 整体解析逻辑失效 |
| 异步加载方式变更 | 低（技术升级） | 等待策略失效 |
| 数据字段位置变更 | 中 | 提取逻辑失效 |

### 1.2 全国主要平台现状

- **重庆市公共资源交易中心**：列表页用表格结构，近期已发生变化（我们踩坑）
- **四川省公共资源交易中心**：动态加载，需JS渲染
- **广东省公共资源交易中心**：API接口驱动，可尝试直接调用
- **贵州省公共资源交易中心**：混合渲染，部分内容需二次请求

## 二、反爬策略识别与应对

### 2.1 常见反爬手段

| 反爬类型 | 特征 | 应对方案 |
|---------|------|---------|
| Cloudflare/WAF | 5秒盾、JS Challenge | puppeteer-stealth、延迟访问 |
| 验证码（滑块/图片） | 弹窗验证 | OCR识别、打码平台、降低频率 |
| IP频率限制 | 连续访问返回403/429 | 代理池、请求间隔随机化 |
| User-Agent检测 | 空UA或异常UA被拒 | 伪装真实UA轮换 |
| Cookie/Session跟踪 | 首次访问种cookie | 复用浏览器实例 |
| 动态Token | 每次请求需携带签发token | 拦截API请求获取token |
| 数据加密传输 | 返回内容为加密串 | JS逆向解密（复杂度高） |

### 2.2 我们的招标爬虫面临的问题

当前 scrape.js 使用的 `table tbody tr` 选择器：
- ✅ 能找到14个元素（说明表格结构存在）
- ❌ 但无法匹配到项目字段（说明字段结构已变）

**可能原因**：
1. 表格列顺序变更
2. 项目名称/金额等字段的CSS类名变化
3. 数据从td直接展示变为td内的嵌套结构
4. 部分字段改为异步加载

## 三、选择器可靠性对比

### 3.1 三种选择器策略对比

| 策略 | 可靠性 | 维护成本 | 性能 | 适用场景 |
|------|-------|---------|------|---------|
| **CSS选择器** | ⭐⭐⭐ | 低 | 高 | 结构稳定页面 |
| **XPath** | ⭐⭐⭐⭐ | 中 | 中 | 复杂层级、文本匹配 |
| **文本/正则匹配** | ⭐⭐⭐⭐⭐ | 高 | 低 | 关键字段提取 |

### 3.2 Playwright内置选择器（推荐）

Playwright提供了比CSS/XPath更健壮的定位方式：

```javascript
// 1. text选择器 — 最抗结构变化
await page.locator('text=招标公告').click();
await page.locator('td:has-text("万元")').textContent();

// 2. 角色选择器 — 基于ARIA角色
await page.locator('role=row').locator('role=cell').nth(2);

// 3. 组合选择器 — 多条件匹配
await page.locator('tr >> td:has-text("2026") >> nth=0');
await page.getByRole('table').getByRole('row').nth(1);
```

**关键优势**：text选择器不依赖CSS类名和DOM层级，只关注可见文本内容。

**来源**：[Web Scraping with Playwright: A Complete Guide](https://software-testing-tutorials-automation.com/2025/09/web-scraping-with-playwright.html)

## 四、Playwright/Puppeteer健壮性方案

### 4.1 多层选择器回退策略

```javascript
// 从最精确到最宽松逐级回退
async function extractProjectName(row) {
  const strategies = [
    () => row.locator('td.project-name a').textContent(),       // 精确CSS
    () => row.locator('td:nth-child(2) a').textContent(),       // 位置CSS
    () => row.locator('td:has(> a)').first().textContent(),    // 语义CSS
    () => row.locator('a:has-text("公告")').textContent(),      // 文本匹配
    () => row.locator('a').first().textContent(),              // 最宽松
  ];
  
  for (const fn of strategies) {
    try {
      const text = await fn();
      if (text && text.trim().length > 5) return text.trim();
    } catch (e) { /* 继续尝试下一个策略 */ }
  }
  return null; // 所有策略失败
}
```

### 4.2 等待策略

```javascript
// ❌ 脆弱：固定等待
await page.waitForTimeout(3000);

// ✅ 健壮：等待网络空闲
await page.waitForLoadState('networkidle');

// ✅ 健壮：等待特定元素
await page.waitForSelector('table tbody tr', { timeout: 30000 });

// ✅ 最健壮：等待条件满足
await page.waitForFunction(() => {
  return document.querySelectorAll('table tbody tr').length > 0;
});
```

### 4.3 页面结构变化检测

```javascript
// 在爬取前先做"健康检查"
async function healthCheck(page) {
  const expectedElements = [
    { selector: 'table', desc: '表格容器' },
    { selector: 'tbody', desc: '表格主体' },
    { selector: 'tr', desc: '数据行' },
  ];
  
  for (const el of expectedElements) {
    const count = await page.locator(el.selector).count();
    if (count === 0) {
      throw new Error(`页面结构变化：找不到 ${el.desc}（${el.selector}）`);
    }
  }
}
```

**来源**：[Scraping Dynamic Websites with Scrapy Playwright](https://scrapfly.io/blog/posts/web-scraping-dynamic-websites-with-scrapy-playwright)

## 五、选择器维护最佳实践

### 5.1 配置化选择器（核心建议）

将所有选择器从代码中抽离到配置文件：

```json
// selectors.json
{
  "site": "重庆市公共资源交易中心",
  "version": "2026-03-23",
  "list": {
    "container": "table tbody",
    "row": "tr",
    "fields": {
      "title": ["td:nth-child(2) a", "td:has(a) a:first-child"],
      "date": ["td:nth-child(5)", "td:has-text('202')"],
      "category": ["td:nth-child(3)", "td .tag"]
    }
  },
  "fallback": {
    "enabled": true,
    "textMatch": {
      "title": "/公告|招标|采购/",
      "amount": "/\\d+\\.?\\d*万元/"
    }
  }
}
```

### 5.2 结构变化告警机制

```javascript
// 爬取完成后做数据完整性检查
function validateResults(projects) {
  const warnings = [];
  
  if (projects.length === 0) {
    warnings.push('🚨 未抓取到任何项目，页面结构可能已变化');
  }
  
  // 检查关键字段完整性
  const missingTitle = projects.filter(p => !p.title || p.title.length < 5).length;
  if (missingTitle > projects.length * 0.3) {
    warnings.push(`⚠️ ${missingTitle}/${projects.length} 项目缺少标题`);
  }
  
  return warnings;
}
```

### 5.3 版本化页面快照

每次成功爬取后保存关键DOM快照：
```javascript
// 保存页面结构摘要（非完整HTML，用于对比）
const snapshot = await page.evaluate(() => {
  return {
    tableCount: document.querySelectorAll('table').length,
    rowCount: document.querySelectorAll('table tbody tr').length,
    columnCount: document.querySelectorAll('table tbody tr td').length / (document.querySelectorAll('table tbody tr').length || 1),
    columnHeaders: Array.from(document.querySelectorAll('table thead th')).map(th => th.textContent.trim()),
    timestamp: new Date().toISOString()
  };
});
```

## 六、重庆市公共资源交易中心适配建议

### 6.1 诊断步骤

1. **打开页面手动检查**：用浏览器开发者工具查看当前表格结构
2. **对比旧选择器**：确认 `table tbody tr` 找到的14个元素是什么
3. **检查字段位置**：项目名称、日期、金额现在在哪个 td/嵌套结构中
4. **测试API接口**：用Network面板查看是否有更稳定的数据接口

### 6.2 具体修复建议

```
优先级排序：
1. 🔍 检查是否存在数据API（最稳定的方案）
2. 📝 更新CSS选择器（中等维护成本）
3. 🔧 改用Playwright text选择器（最抗变化）
4. 🛡️ 添加健康检查和告警（防止再次静默失败）
5. 📸 实现页面快照对比（快速定位变化）
```

### 6.3 长期方案

考虑将招标数据源从单一平台扩展到多平台聚合：
- API接口直连（最稳定）
- 多平台交叉验证（防单点故障）
- 定期选择器健康检查（自动告警）

## 八、学习总结

**核心收获**：

1. **选择器配置化 > 硬编码**：将选择器抽离为JSON配置，更新时只改配置不改代码
2. **text选择器 > CSS选择器**：Playwright的text/role选择器抗页面结构变化能力远强于CSS
3. **多层回退 > 单一策略**：设计3-5级选择器回退链，一级失败自动尝试下一级
4. **健康检查 + 告警 > 静默失败**：爬取前检查页面结构，爬取后验证数据完整性
5. **API优先 > 页面解析**：优先寻找数据API接口，最稳定
