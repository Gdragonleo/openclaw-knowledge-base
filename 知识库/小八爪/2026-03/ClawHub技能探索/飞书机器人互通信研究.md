# 飞书机器人互通信研究 - 完整方案

**创建时间**：2026-03-10 23:54
**研究目标**：小八爪和小鲸鱼如何在飞书中互相通信

---

## 🎯 可行方案

### 方案1：通过OpenClaw的sessions_send（✅ 推荐）

**原理**：
- OpenClaw支持跨session通信
- 使用 `sessions_send(sessionKey, message)` 
- 可以给其他智能体发送消息

**步骤**：
1. 获取小鲸鱼的sessionKey
2. 小八爪使用 `sessions_send` 发送 `[BOT_TASK]`
3. 小鲸鱼在session中收到消息并处理
4. 小鲸鱼使用 `sessions_send` 回复 `[BOT_REPLY]`

**代码示例**：
```javascript
// 小八爪发送任务给小鲸鱼
sessions_send({
  sessionKey: "agent:main:feishu:direct:小鲸鱼的user_id",
  message: "[BOT_TASK] 请帮我分析这张图片"
})

// 小鲸鱼回复
sessions_send({
  sessionKey: "agent:main:feishu:direct:小八爪的user_id",
  message: "[BOT_REPLY] 图片分析结果：..."
})
```

### 方案2：通过共享飞书群（✅ 可行）

**原理**：
- 两个机器人都在同一个飞书群
- 通过群聊@对方

**步骤**：
1. 创建协作群（小八爪 + 小鲸鱼 + 小刘）
2. 小八爪在群里 @小鲸鱼 发送 `[BOT_TASK]`
3. 小鲸鱼看到后回复 `[BOT_REPLY]`

**代码示例**：
```javascript
// 小八爪在群里发送
message({
  action: "send",
  channel: "feishu",
  target: "oc_协作群ID",
  message: "@小鲸鱼 [BOT_TASK] 请帮我分析图片"
})

// 小鲸鱼在群里回复
message({
  action: "send",
  channel: "feishu",
  target: "oc_协作群ID",
  message: "[BOT_REPLY] 分析结果：..."
})
```

---

### 方案3：通过共享文件（⚠️ 间接方式）

**原理**：
- 两个机器人共享同一个文件系统
- 通过读写文件实现通信

**步骤**：
1. 小八爪写任务到 `协作/任务.md`
2. 小鲸鱼定时读取 `协作/任务.md`
3. 小鲸鱼执行并写结果到 `协作/回复.md`
4. 小八爪读取 `协作/回复.md`

**缺点**：
- ⏱️ 有延迟（需要定时检查）
- 🔄 不够实时

## ✅ 推荐方案：sessions_send

**为什么推荐？**
1. ✅ 最符合OpenClaw设计
2. ✅ 实时、可靠
3. ✅ 不需要额外配置
4. ✅ 支持私聊协作

**需要的信息**：
- 小鲸鱼的sessionKey（格式：`agent:main:feishu:direct:{user_id}`）
- 或者小鲸鱼的open_id
