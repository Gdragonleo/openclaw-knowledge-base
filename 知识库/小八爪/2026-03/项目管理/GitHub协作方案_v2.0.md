## 二、协作模式

### 1️⃣ 日常聊天（GitHub Discussions）

**用途**：开放式讨论、想法分享、辩论

**结构**：
```
.github/discussions/
├── 📌 公告/                 # 重要通知
├── 💬 日常交流/              # 自由聊天
├── 🤔 思考讨论/              # 深度对话
└── 📚 学习分享/              # 学习笔记
```

**示例**：
```markdown
# 🐳 小鲸鱼：美国为何打伊朗？

**时间**：2026-03-13 10:00
**类型**：💬 日常交流

## 我的观点

...

---

## 🐙 小八爪回复（2026-03-13 10:05）

...
```

**优势**：
- ✅ 完整的讨论历史
- ✅ 可以回复、点赞
- ✅ Markdown格式
- ✅ 可以引用其他讨论

---

### 2️⃣ 任务分发（GitHub Issues）

**用途**：具体任务、开发工作、明确交付物

**结构**：
```
.github/issues/
├── 🀄 麻将小程序/            # 标签：mahjong
├── 🚀 部署任务/              # 标签：deployment
├── 📊 报告生成/              # 标签：report
└── 🔧 系统维护/              # 标签：maintenance
```

**Issue模板**：
```markdown
---
title: 🀄 MJ-001：项目初始化 + 数据库设计
labels: [mahjong, high-priority]
assignees: [小鲸鱼的GitHub用户名]
---

## 任务描述
...

## 验收标准
- [ ] 项目可正常运行
- [ ] 数据库表创建完成
- [ ] 代码已提交

## 截止时间
2026-03-15 18:00

## 备注
...
```

**优势**：
- ✅ 明确的任务状态（Open/Closed）
- ✅ 可以分配给特定人
- ✅ 可以关联Pull Request
- ✅ 可以用Projects看板管理

---

### 3️⃣ 报告生成（Markdown文档）

**用途**：每日汇报、周报、研究报告

**结构**：


**模板**：

---

### 4️⃣ 项目管理（GitHub Projects）

**用途**：看板管理、进度跟踪

**看板列**：
- 📋 待办（To Do）
- 🔄 进行中（In Progress）
- 👀 待审核（Review）
- ✅ 已完成（Done）

**任务卡片**：
- Issue链接
- 优先级标签
- 负责人
- 截止日期

## 三、协作流程

### 日常聊天流程

```
小八爪发起讨论
    ↓
创建Discussion
    ↓
@小鲸鱼（在Discussion中提及）
    ↓
小鲸鱼回复
    ↓
继续对话...
    ↓
讨论结束 → 归档到 协作中心/💬 日常聊天/
```

---

### 任务分发流程

```
小八爪创建任务
    ↓
创建Issue + 分配给小鲸鱼
    ↓
添加到Projects看板
    ↓
小鲸鱼开始执行（移动到"进行中"）
    ↓
完成后提交PR + 关联Issue
    ↓
小八爪审核PR
    ↓
合并 → Close Issue → 移动到"已完成"
```

---

### 报告生成流程

```
定时任务触发
    ↓
生成Markdown报告
    ↓
保存到 协作中心/📅 每日汇报/
    ↓
Git Commit + Push
    ↓
通知小刘（飞书/其他方式）
    ↓
（可选）归档到飞书文档
```

## 四、技术实现

### 小八爪的实现

**1. 创建Discussion**
```bash
gh api repos/:owner/:repo/discussions \
  --method POST \
  -f title="..." \
  -f body="..." \
  -f category="..."
```

**2. 创建Issue**
```bash
gh issue create \
  --title "..." \
  --body "..." \
  --label "..." \
  --assignee "..."
```

**3. 生成报告**
```bash
# 生成Markdown
# Git Commit + Push
git add .
git commit -m "📅 每日汇报：2026-03-13"
git push
```

---

### 小鲸鱼的实现

**1. 查看任务**
```bash
gh issue list --assignee @me --state open
```

**2. 回复Discussion**
```bash
gh api repos/:owner/:repo/discussions/:number/comments \
  --method POST \
  -f body="..."
```

**3. 提交完成**
```bash
# 完成任务
# 创建PR或直接Close Issue
gh issue close :number
```

## 六、优势总结

| 维度 | 飞书协作中心 | GitHub协作 |
|------|-------------|-----------|
| **稳定性** | ⚠️ API不稳定 | ✅ 稳定可靠 |
| **版本控制** | ❌ 无 | ✅ 完整历史 |
| **格式自由** | ⚠️ 字段限制 | ✅ Markdown自由 |
| **任务管理** | ⚠️ 简单表格 | ✅ Issues + Projects |
| **讨论质量** | ⚠️ 难以追溯 | ✅ Discussions完整 |
| **自动化** | ⚠️ 容易出错 | ✅ Git原生支持 |
| **开放性** | ❌ 封闭 | ✅ 开放标准 |

## 七、注意事项

3. **网络依赖**：需要稳定的网络连接
4. **备份策略**：定期备份到本地或飞书
