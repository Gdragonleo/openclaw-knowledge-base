## 一、核心概念总结

### 1.1 任务管理系统选择

项目管理的基础是**单一信息源（Single Source of Truth）**。

**系统对比**：
| 系统 | 适用场景 | 复杂度 | 成本 |
|------|---------|--------|------|
| 笔记本/子弹笔记 | 模拟爱好者、极简工作流 | 极低 | $5 |
| 待办清单（Todoist, Things） | 个人任务、简单流程 | 低 | 免费-$5/月 |
| 看板（Trello, Notion） | 视觉思考者、多阶段流程 | 低-中 | 免费-$10/月 |
| 项目工具（Asana, ClickUp） | 多项目、客户协作 | 中-高 | 免费-$20/月 |
| 电子表格（Sheets, Airtable） | 自定义流程、数据驱动 | 中 | 免费 |

**核心原则**：从简单开始，当简单不再适用时再增加复杂度。

---

### 1.2 工作-项目-任务层级结构

```
领域（AREA）—— 宽泛的生活/业务域
  ↓
项目（PROJECT）—— 有明确终点的目标
  ↓
任务（TASK）—— 单次会话可完成的行动
  ↓
子任务（SUBTASK）—— 任务拆解的最小步骤
```

**关键规则**：
- 项目有清晰的"完成状态"（Done = 明确的终点）
- 任务是单一行动，不是项目
- 感到overwhelmed的任务需要拆解成子任务
- **个体创业者同时活跃项目不超过3-5个**

---

### 1.3 艾森豪威尔矩阵优先级管理

```
              紧急
         |           |
重要     | Q1 立即做  | Q2 计划做
         |           |
---------|-----------|----------
不重要   | Q3 委托做  | Q4 删除
         |           |
```

**四象限策略**：
- **Q1（紧急+重要）**：危机、deadline、关键bug → 立即处理
- **Q2（不紧急+重要）**：战略工作、规划、技能提升 → **排定时间，这是成长的核心**
- **Q3（紧急+不重要）**：打断、低价值请求 → 委托或拒绝
- **Q4（不紧急+不重要）**：忙碌工作、干扰 → 删除

**关键洞察**：
> 大多数个体创业者在Q1和Q3投入过多，在Q2投入不足。**战略工作（Q2）才是业务增长的关键**。

### 1.4 周计划仪式（周日或周一）

**每周15-30分钟，四步流程**：

**Part 1: 复盘上周**
- ✅ 完成了什么？（庆祝胜利）
- ❌ 没完成什么？（为什么？重新排期或删除？）
- 🚧 有什么阻碍？

**Part 2: 设定本周Top 3优先级**
- 不是任务，是**成果**
- 错误示例："做营销"
- 正确示例："发布2篇博客+安排5条社交媒体帖子"

**Part 3: 时间锁定大石头**
- 为每个优先级锁定具体的日历时间
- 例：周一9-11点写博客1，周二9-11点写博客2

**Part 4: 大脑清空和分类**
- 列出其他所有需要关注的事项
- 分类为：本周 / 下周 / 未来某天

**黄金法则**：不排进日历的工作不会发生。**先锁定Top 3优先级，其他工作围绕它们安排。**

---

### 1.5 日计划仪式

**每天5-10分钟，四个步骤**：

1. **查日历**：今天有什么已安排的事项？
2. **回顾周优先级**：今天最该推进的重要事项是什么？
3. **选出1-3个任务**：每日任务不超过3个，否则注定失败
4. **时间锁定**：为每个任务分配具体时间段

**日任务清单结构**：
```
今日最高优先级：[必须完成的那个任务]
  时间：[什么时候做]

次要任务：
  - [任务2] — 时间：[何时]
  - [任务3] — 时间：[何时]

如有时间：
  - [锦上添花的任务]
  - [低优先级任务]
```

**日终仪式（5分钟）**：
- ✅ 标记完成的任务（多巴胺奖励 + 可视化）
- ⏭️ 未完成任务移到明天或更晚
- 📝 记录阻碍或胜利

---

### 1.6 多阶段项目管理

**2周以上的大型项目，分阶段管理**：

```
项目：[名称]
目标：[成功是什么样子？]
截止日期：[目标完成日期]
负责人：[你或委托人]

阶段：
  阶段1：[名称]（目标：第1周）
    - 任务1
    - 任务2
    - 任务3

  阶段2：[名称]（目标：第2周）
    - 任务4
    - 任务5

  阶段3：[名称]（目标：第3周）
    - 任务6
    - 任务7

状态：[未开始 / 进行中 / 受阻 / 完成]
阻碍：[什么阻止了进展？]
备注：[背景、链接、决策]
```

**每周项目检查**：
- 当前在哪个阶段？
- 能按时完成吗？
- 有阻碍吗？（如有，升级或解决）

---

### 1.7 客户项目管理

**与客户或承包商协作时，可视化是关键**。

**客户项目看板结构（Trello或Notion）**：
```
列：
  - 待办（未开始的任务）
  - 进行中（你正在做的）
  - 审核（等待客户反馈）
  - 完成（已完成且批准）

卡片结构（每任务）：
  - 标题：[交付物名称]
  - 截止日期：[何时]
  - 分配给：[你或承包商]
  - 描述：[需要交付什么]
  - 附件：[文件、链接、参考资料]
  - 清单：[子步骤]
  - 评论：[客户反馈、备注]
```

**客户沟通规则**：
- 每周更新项目状态（简短邮件或分享看板链接）
- 准备好反馈时移到"审核"列
- 审核列任务不超过3天没有跟进

---

### 1.8 进度追踪和复盘

**月度复盘（30分钟）**：
- [ ] 本月完成了多少项目？
- [ ] 周优先级完成率是多少？（目标70-80%）
- [ ] 什么比预期花了更长时间？（为什么？）
- [ ] 哪些工具或流程拖慢了你？（修复它们）
- [ ] 应该停止做什么？（低价值工作、干扰）
- [ ] 应该开始做什么？（委托、自动化、新系统）

**关键指标（可选但有价值）**：
- 每周完成任务数（看趋势）
- Q2战略工作 vs Q1反应性工作的时间比例
- 每季度完成项目数
- deadline达成率

**黄金法则**：如果持续错过deadline或优先级，说明系统不工作。简化或调整。

### 2.1 麻将团队（产品开发导向）

**特点**：
- 产品迭代快
- 需求变化频繁
- 技术任务可拆解
- 有明确的版本发布节点

**推荐系统**：**Notion + 看板**

**项目示例**：
```
领域：麻将小程序
  项目：V2.0 版本发布
    目标：完成3个核心功能上线
    截止：2026-04-15
    
    阶段1：需求确认（Week 1）
      - 用户调研报告
      - 功能优先级排序
      - 技术可行性评估
    
    阶段2：开发测试（Week 2-3）
      - UI设计稿
      - 前端开发
      - 后端API
      - 单元测试
    
    阶段3：上线部署（Week 4）
      - 灰度测试
      - Bug修复
      - 正式发布
```

**艾森豪威尔矩阵应用**：
- Q1：线上bug、用户投诉、deadline倒计时
- Q2：新功能规划、技术架构优化、用户增长策略
- Q3：第三方工具对接、运营活动支持
- Q4：无意义会议、过度优化

**周计划示例**：
- Top 1：完成支付功能开发
- Top 2：用户调研收集20份问卷
- Top 3：修复已知bug清单

**日计划示例**：
- 最高优先级：完成支付接口调试（9:00-12:00）
- 次要任务：回复用户反馈（14:00-15:00）
- 如有时间：整理技术文档

### 2.2 科研团队（学术研究导向）

**特点**：
- 项目周期长（月/季度级别）
- 需要深度思考和文献调研
- 阶段性成果（论文、实验）
- 时间灵活但容易拖延

**推荐系统**：**子弹笔记 + Google Calendar**

**项目示例**：
```
领域：学术研究
  项目：论文《AI驱动的知识管理系统》
    目标：投稿到IEEE会议
    截止：2026-06-01
    
    阶段1：文献调研（Week 1-2）
      - 阅读相关论文20篇
      - 整理研究空白
      - 确定研究问题
    
    阶段2：方法设计（Week 3-4）
      - 设计实验框架
      - 数据收集方案
      - 评估指标
    
    阶段3：实验与分析（Week 5-8）
      - 实现原型系统
      - 运行实验
      - 数据分析
    
    阶段4：论文撰写（Week 9-12）
      - Introduction
      - Methodology
      - Results
      - Conclusion
      - 修改润色
```

**艾森豪威尔矩阵应用**：
- Q1：会议deadline、合作者等待的交付物
- Q2：**深度思考、方法论创新、文献阅读、实验设计**
- Q3：行政事务、临时邀约
- Q4：无意义的学术社交、过度阅读（超过需求）

**周计划示例**：
- Top 1：完成Related Work章节初稿
- Top 2：运行第一组实验
- Top 3：与导师会面确认方向

**日计划示例**：
- 最高优先级：撰写Related Work第2节（9:00-12:00，13:30-15:30）
- 次要任务：整理实验数据（16:00-17:00）
- 如有时间：阅读2篇相关论文

**特殊建议**：
- 科研需要大量Q2时间（深度思考），**坚决保护整块时间**
- 使用时间锁定避免"一直在读但没输出"
- 每周必须有可交付成果（哪怕很小）

**艾森豪威尔矩阵应用**：

- 如有时间：更新项目看板

**特殊建议**：
- 多客户并行时，**每个客户有独立的看板**
- 使用"本周计划"列明确当前焦点
- Q1工作占比高是正常的，但要每周投入Q2（方法论沉淀）

## 三、常见错误及规避

### 3.1 缺乏单一信息源
- **问题**：任务散落在邮件、笔记本、Slack、脑子里 → 混乱
- **解决**：选择**一个**系统，只使用它

### 3.2 系统过度复杂
- **问题**：PM工具每天需要15分钟维护 → 太复杂
- **解决**：保持简单，能用就行

### 3.3 没有周计划
- **问题**：每天都在"救火"但没有战略进展
- **解决**：每周日/周一做15分钟周计划

### 3.4 活跃项目过多
- **问题**：10个并行项目 = 0个完成项目
- **解决**：限制同时活跃项目≤3-5个

### 3.5 任务太大不拆解
- **问题**："上线产品"太overwhelming，永远不开始
- **解决**：拆解成"写落地页文案"这样的可执行任务

### 3.6 不懂得拒绝
- **问题**：每个Yes都是对其他事情的No
- **解决**：**无情地保护优先级**
