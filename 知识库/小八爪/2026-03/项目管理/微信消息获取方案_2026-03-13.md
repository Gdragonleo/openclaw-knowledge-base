```python
cursor.execute("SELECT content FROM MSG WHERE talker = ? ORDER BY createTime DESC", ("联系人ID",))
messages = cursor.fetchall()

# 4. 解密（微信数据库有加密）
# 需要获取密钥（从内存或配置文件提取）
```

**优点**：
- ✅ 可以获取所有历史消息
- ✅ 数据完整（文字、图片、文件、语音）
- ✅ 不依赖UI

**缺点**：

### 方案2：截屏OCR识别 ⭐⭐⭐⭐（实用派）

**原理**：定期截屏 → OCR识别文字

**实现**：
```python
import pyautogui
import pytesseract
from PIL import Image

# 1. 截取微信窗口
wechat_window = get_wechat_window_position()
screenshot = pyautogui.screenshot(region=wechat_window)

# 2. OCR识别文字
text = pytesseract.image_to_string(screenshot, lang='chi_sim')

# 3. 解析消息内容
messages = parse_messages(text)
```

**优点**：
- ✅ 不需要破解数据库
- ✅ 技术简单
- ✅ 可以监控多个聊天

**缺点**：
- ❌ 只能获取当前显示的消息
- ❌ 需要微信窗口在前台
- ❌ OCR可能有误差
- ❌ 不能获取历史消息

**适用场景**：
- 监控特定聊天的实时消息
- 提取屏幕上的关键信息

### 方案3：企业微信机器人回调 ⭐⭐⭐⭐⭐（官方推荐）

**原理**：配置机器人回调URL，接收消息推送

**步骤**：
```
1. 企业微信管理后台 → 应用管理 → 创建应用
2. 设置接收消息的URL（需要公网服务器）
3. 配置消息回调
4. 微信服务器会将消息POST到你的URL
```

**代码示例**：
```python
from flask import Flask, request

app = Flask(__name__)

@app.route('/wechat/callback', methods=['POST'])
def receive_message():
    # 接收微信推送的消息
    message = request.json
    
    # 解析消息内容
    msg_type = message['MsgType']
    content = message['Content']
    sender = message['FromUserName']
    
    # 处理消息
    print(f"收到消息: {content} (来自: {sender})")
    
    return "success"

if __name__ == '__main__':
    app.run(port=80)
```

**优点**：
- ✅ 官方API，稳定可靠
- ✅ 无封号风险
- ✅ 实时推送，无需轮询
- ✅ 可以接收和发送

**缺点**：
- ❌ 需要企业微信（个人微信不支持）
- ❌ 需要公网服务器
- ❌ 配置相对复杂

**适用场景**：
- 企业内部机器人
- 自动客服
- 消息自动处理

### 方案4：AppleScript + 剪贴板 ⭐⭐⭐（曲线救国）

**原理**：用AppleScript全选复制消息到剪贴板，再读取剪贴板

**实现**：
```applescript
tell application "WeChat"
    activate
end tell

delay 1

tell application "System Events"
    tell process "WeChat"
        # 全选当前聊天内容
        keystroke "a" using command down
        delay 0.5
        
        # 复制到剪贴板
        keystroke "c" using command down
        delay 0.5
    end tell
end tell

# 获取剪贴板内容
set messageContent to the clipboard as text

return messageContent
```

**优点**：
- ✅ 可以获取当前聊天的消息
- ✅ 不需要破解数据库
- ✅ 实现简单

**缺点**：
- ❌ 需要微信窗口在前台
- ❌ 会干扰用户操作（全选、复制）
- ❌ 只能获取当前显示的内容
- ❌ 格式可能丢失

**适用场景**：
- 定期备份聊天记录
- 提取当前聊天的关键信息

## ⚠️ 重要提示

**关于AppleScript的限制**：
```
AppleScript只能"控制"应用，不能"读取"数据
要获取消息内容，必须用其他方法
```

**关于封号风险**：
```
✅ 企业微信机器人：无风险（官方API）
⚠️ 数据库提取：有风险（违反协议）
✅ 截屏OCR：无风险（只是截图）
⚠️ 个人号API：高风险（微信禁止）
```

**最佳实践**：
```
1. 企业场景：用企业微信机器人
2. 个人监控：用截屏OCR
3. 数据备份：手动导出
4. 避免：任何自动化个人号微信的方案
```
