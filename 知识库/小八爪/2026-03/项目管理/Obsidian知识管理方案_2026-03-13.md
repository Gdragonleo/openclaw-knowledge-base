## 📋 Obsidian核心理念（采用）

### **1. 双向链接系统** ⭐⭐⭐⭐⭐

#### **使用方式**：
```markdown
<!-- 传统写法 -->

招标系统相关内容在 [[招标系统]] 中，使用 [[爬虫]] 技术，参考 [[Python脚本]]
```

#### **立即生效**：
```
✅ 所有内部引用改为 [[文件名]]
✅ 自动形成知识网络
✅ 便于快速跳转
```

---

### **2. 原子化笔记** ⭐⭐⭐⭐⭐

#### **原则**：
```
❌ 一个文件包含所有内容
✅ 一个文件一个主题（原子化）

示例：
❌ 招标系统完整文档.md（100KB）
✅ 招标系统概述.md（5KB）
✅ 招标爬虫设计.md（10KB）
✅ 招标报告格式.md（8KB）
```

#### **优势**：
```
✅ 易于链接
✅ 易于维护
✅ 易于查找
✅ 易于复用
```

---

### **3. MOC（Map of Content）** ⭐⭐⭐⭐⭐

#### **概念**：
```
MOC = 知识地图/索引页
一个主题的入口，链接到所有相关笔记
```

#### **示例**：
```markdown
# 招标系统 MOC

## 概述
- [[招标系统概述]]
- [[招标系统发展历程]]

## 技术实现
- [[招标爬虫设计]]
- [[数据清洗流程]]
- [[报告生成算法]]

## 运营
- [[每日招标报告]]
- [[招标报告定制化]]
- [[招标数据统计]]

## 相关项目
- [[刘氏集团业务]]
- [[小八爪能力清单]]
```

---

### **4. 每日笔记（Daily Notes）** ⭐⭐⭐⭐

内容：
- 今日完成
- 今日思考

### **5. 标签系统** ⭐⭐⭐⭐

#### **使用方式**：
```markdown
#项目/招标系统 #技术/爬虫 #状态/进行中
```

#### **标签分类**：
```

#状态/ - 进行状态
#类型/ - 文档类型
#人物/ - 相关人物
```
