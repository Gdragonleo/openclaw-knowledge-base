## 📋 新增工作流程

### **流程1：错误预防检查** ⭐⭐⭐⭐⭐

#### **触发时机**：每次接收用户消息时

#### **执行步骤**：
```
1. 接收消息
   ↓
2. 提取关键词（主题、操作、对象）
   ↓
3. 查询错误库（skills/error-lessons/lessons/）
   ↓
4. 匹配相关教训（标签匹配）
   ↓
5. 应用预防规则
   ↓
6. 执行任务
   ↓
7. 验证结果
```

#### **具体实现**：

##### **步骤1：提取关键词**
```python
# 示例
用户消息："帮我配置飞书权限"
关键词：['飞书', '权限', '配置']
```

##### **步骤2：查询错误库**
```bash
# 在错误库中搜索相关教训
grep -r "飞书\|权限" skills/error-lessons/lessons/
```

##### **步骤3：应用预防规则**
```
发现教训5：飞书权限配置指导错误
预防规则：
- 遇到飞书问题，必须先查阅API文档
- 不确定的操作指导，先说明"需要确认"
- 技术类问题，优先查找官方文档
```

##### **步骤4：执行并验证**
```
行动：
1. 先查阅飞书开放平台API文档
2. 确认正确的配置路径
3. 给用户提供准确指导
4. 添加验证步骤（"请截图确认是否成功"）
```

---

### **流程2：信息验证机制** ⭐⭐⭐⭐⭐

#### **触发时机**：回答技术问题、配置指导、决策建议时

#### **执行步骤**：
```
1. 接收问题
   ↓
2. 识别问题类型
   - 技术配置类
   - 决策建议类
   - 知识查询类
   ↓
3. 搜索验证
   - 官方文档（优先级1）
   - 百度/Google（优先级2）
   - 已有知识库（优先级3）
   ↓
4. 分析对比
   ↓
5. 给出答案（注明信息来源）
   ↓
6. 标记置信度
```

#### **问题分类和处理**：

##### **类型1：技术配置类**（必须验证）
```
示例：服务器部署、npm安装、Git配置
要求：
- 必须查官方文档
- 必须验证命令语法
- 必须标注来源链接
```

##### **类型2：决策建议类**（必须验证）
```
示例：技术选型、方案对比、成本估算
要求：
- 必须搜索最新信息
- 必须提供对比表格
- 必须标注数据来源
```

##### **类型3：知识查询类**（建议验证）
```
示例：概念解释、背景介绍
要求：
- 建议搜索验证
- 如不确定，明确说明"需要确认"
```

## 🔧 具体实现工具

### **工具1：错误库查询命令**
```bash
# 查询相关教训
function check_errors() {
  local keywords=$1
  echo "🔍 检查错误库：$keywords"
  
  # 搜索教训
  grep -r "$keywords" ~/.openclaw/workspace/skills/error-lessons/lessons/
  
  # 如果找到，显示预防规则
  if [ $? -eq 0 ]; then
    echo "⚠️ 发现相关教训，已应用预防规则"
  fi
}
```

### **工具2：信息验证流程**
```bash
# 验证技术信息
function verify_info() {
  local topic=$1
  local source_type=$2  # official/baidu/knowledge
  
  echo "🔍 验证信息：$topic"
  
  case $source_type in
    "official")
      # 优先级1：官方文档
      echo "📖 查阅官方文档..."
      ;;
    "baidu")
      # 优先级2：百度搜索
      echo "🔎 百度搜索..."
      ;;
    "knowledge")
      # 优先级3：已有知识
      echo "📚 查阅知识库..."
      ;;
  esac
}
```

## 📊 工作流程示例

### **示例1：回答服务器配置问题**

#### **旧流程**（容易出错）：
```
用户："服务器上怎么安装OpenClaw？"
→ 我凭记忆回答
→ 可能版本错误
→ 可能命令错误
→ 用户执行失败
```

#### **新流程**（验证后回答）：
```
用户："服务器上怎么安装OpenClaw？"
   ↓
1. 检查错误库
   - 关键词：['服务器', '安装', 'OpenClaw']
   - 发现已安装错误：Node.js版本问题
   - 应用预防规则：必须使用Node.js 18+
   ↓
2. 搜索验证
   - 查阅OpenClaw官方文档
   - 查阅Node.js官方文档
   - 对比百度最新教程
   ↓
3. 给出答案（标注来源）
   "根据OpenClaw官方文档[链接]，安装步骤如下：
    1. 安装Node.js 18+（验证过）
    2. npm install -g openclaw
    注意：Node.js版本必须>=18.0.0"
   ↓
4. 添加置信度
   "✅ 信息已验证，置信度：95%"
```

## 📝 置信度评分标准

### **95-100%**：完全确认
- ✅ 官方文档验证
- ✅ 实际测试通过
- ✅ 多个来源一致

### **80-95%**：高度确认
- ✅ 官方文档验证
- ⚠️ 未实际测试

### **60-80%**：中度确认
- ✅ 第三方来源验证
- ⚠️ 缺少官方确认

### **40-60%**：低度确认
- ⚠️ 仅凭记忆或经验
- ⚠️ 建议用户验证

### **< 40%**：不确定
- ❌ 明确告知"需要确认"
- ❌ 建议查阅官方文档
